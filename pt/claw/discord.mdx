Conecte sua instância Claw ao Discord e converse com a IA diretamente em seu servidor ou DMs.

## Pré-requisitos

- Uma conta no [Discord](https://discord.com/)
- Um servidor Discord onde você tenha permissão de **Gerenciar Servidor**
- Uma instância Claw ([crie uma aqui](https://lemondata.cc/dashboard/claw))

## Guia de Configuração

<Steps>
  <Step title="Criar uma Aplicação Discord">
    1. Vá para o [Discord Developer Portal](https://discord.com/developers/applications)
    2. Clique em **New Application**
    3. Dê um nome a ela (ex: "Meu Bot de IA") e clique em **Create**
  </Step>
  <Step title="Criar um Bot">
    1. Em sua aplicação, vá para a aba **Bot**
    2. Clique em **Add Bot** → **Yes, do it!**
    3. Em **Privileged Gateway Intents**, habilite:
       - **Message Content Intent** (obrigatório)
       - **Server Members Intent** (recomendado)
    4. Clique em **Reset Token** para gerar um token de bot
    5. Copie o token — você precisará dele a seguir
    <Warning>
    O token é exibido apenas uma vez. Se você o perder, precisará redefini-lo.
    </Warning>
  </Step>
  <Step title="Convidar o Bot para seu Servidor">
    1. Vá para a aba **OAuth2** → **URL Generator**
    2. Em **Scopes**, selecione: `bot`, `applications.commands`
    3. Em **Bot Permissions**, selecione: `Send Messages`, `Read Message History`
    4. Copie a URL gerada e abra-a em seu navegador
    5. Selecione seu servidor e clique em **Authorize**
  </Step>
  <Step title="Configurar no Dashboard da LemonData">
    1. Vá para o seu [Claw Dashboard](https://lemondata.cc/dashboard/claw)
    2. Clique no **ícone de configurações** em sua instância
    3. Abra a aba **IM Integrations**
    4. Expanda a seção **Discord**
    5. Cole seu **Bot Token**
    6. Clique em **Save**
  </Step>
  <Step title="Começar a Conversar">
    1. Em seu servidor Discord, mencione o bot ou envie uma DM para ele
    2. Conclua o processo de **pareamento** (insira o código em sua Claw WebUI)
    3. Comece a conversar com a IA!
  </Step>
</Steps>

## Uso em Servidores

### Configurações de Grupo

Configure o comportamento do servidor (guild) na aba **IM Integrations** do seu Dashboard:

| Configuração | Opções | Descrição |
|---------|---------|-------------|
| **Group Policy** | Open / Allowlist / Disabled | Controla o acesso aos canais do servidor |
| **Require @mention** | On / Off | O bot só responde quando @mencionado (recomendado) |
| **Group IDs** | Lista de IDs de guild | Restringir a servidores específicos (vazio = todos os servidores) |

<Note>
O Discord usa listas de usuários permitidos por servidor (guild) em vez de uma lista global de grupo. Para filtragem avançada de usuários por servidor, configure diretamente no arquivo de configuração do OpenClaw.
</Note>

### Política de Mensagens Diretas (DM Policy)

Controle quem pode enviar mensagens diretas ao seu bot:

| Política | Comportamento |
|--------|----------|
| **Pairing** (padrão) | Novos usuários recebem um código de pareamento — insira-o na WebUI para autorizar |
| **Allowlist** | Apenas os IDs de usuário que você especificar podem enviar DMs ao bot |
| **Open** | Qualquer pessoa pode enviar DMs ao bot (use com cautela) |

## Solução de Problemas

<AccordionGroup>
  <Accordion title="O bot está online, mas não responde">
    - Certifique-se de que o **Message Content Intent** está habilitado no Developer Portal
    - Verifique se o token do bot está correto
    - Verifique se o bot tem permissão para ler e enviar mensagens no canal
  </Accordion>
  <Accordion title="O bot aparece como offline">
    - Verifique o status da sua instância no dashboard (deve estar verde/running)
    - Tente reiniciar a instância
    - Verifique se o token não foi redefinido no Developer Portal
  </Accordion>
  <Accordion title="Erros 409 Conflict nos logs">
    - Isso acontece quando duas instâncias tentam usar o mesmo token de bot
    - Certifique-se de que apenas uma instância Claw esteja configurada com este token
  </Accordion>
</AccordionGroup>
