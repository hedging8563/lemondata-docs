## Visão Geral

O Dify é uma plataforma de desenvolvimento de aplicações de LLM de código aberto. Ele fornece orquestração visual de prompts, pipelines de RAG, frameworks de agentes e recursos de LLMOps. O LemonData pode ser configurado como um provedor de modelo personalizado no Dify.

## Benefícios

- Acesse mais de 300 modelos de IA através de uma única interface
- Alterne entre modelos sem alterar a lógica da aplicação
- Otimize custos selecionando o melhor modelo para cada tarefa
- Faturamento unificado e rastreamento de uso

## Pré-requisitos

- Conta LemonData com acesso à API
- Instalação do Dify (nuvem ou self-hosted)

## Passos de Configuração

### Passo 1: Obtenha sua API Key

1. Faça login no [LemonData Dashboard](https://lemondata.cc/dashboard)
2. Navegue até [API Keys](https://lemondata.cc/dashboard/api)
3. Crie e copie sua API key (formato: `sk-...`)

### Passo 2: Adicione um Provedor de Modelo Personalizado

<Steps>
  <Step title="Abra as Configurações">
    No Dify, vá em **Settings** → **Model Provider**
  </Step>
  <Step title="Adicione um Provedor Compatível com OpenAI">
    Clique em **Add Model Provider** e selecione **OpenAI-API-compatible**
  </Step>
  <Step title="Configure o Provedor">
    Insira as seguintes configurações:

    | Campo | Valor |
    |-------|-------|
    | Provider Name | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="Adicione Modelos">
    Adicione os modelos que deseja usar:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### Passo 3: Teste a Conexão

1. Selecione um modelo (ex: `gpt-4o-mini`)
2. Envie uma mensagem de teste
3. Verifique se você recebeu uma resposta

## Uso em Aplicações

### Chatbot

1. Crie uma nova aplicação de Chatbot
2. Selecione o LemonData como o provedor de modelo
3. Escolha seu modelo preferido
4. Configure o prompt de sistema e os parâmetros

### Agente

1. Crie uma aplicação de Agente
2. Selecione um modelo capaz (GPT-4o, Claude)
3. Adicione ferramentas e bases de conhecimento
4. Configure o comportamento do agente

### Workflow

1. Crie um Workflow
2. Adicione nós de LLM
3. Selecione modelos do LemonData para cada nó
4. Conecte os nós e configure o fluxo de dados

## Modelos Disponíveis

| Categoria | Modelos |
|----------|--------|
| Chat | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| Embeddings | text-embedding-3-small, text-embedding-3-large |
| Visão | GPT-4o (com imagens), Claude Sonnet (com imagens) |

## Configuração de RAG

Para aplicações de RAG, configure os embeddings:

1. Vá em **Settings** → **Model Provider**
2. Adicione o modelo de embedding: `text-embedding-3-small`
3. Defina como o modelo de embedding padrão nas configurações da base de conhecimento

## Solução de Problemas

<AccordionGroup>
  <Accordion title="Erro de Conexão">
    - Verifique se a API Base URL é exatamente `https://api.lemondata.cc/v1`
    - Verifique se há barras no final (trailing slashes)
    - Verifique a conectividade da rede
  </Accordion>

  <Accordion title="401 Unauthorized">
    - Verifique novamente a API key
    - Certifique-se de que a chave está ativa no dashboard do LemonData
  </Accordion>

  <Accordion title="Modelo Não Encontrado">
    - Verifique se o nome do modelo corresponde exatamente
    - Verifique a disponibilidade do modelo em [lemondata.cc/en/models](https://lemondata.cc/pt/models)
  </Accordion>
</AccordionGroup>
