---
title: "Gerar Conteúdo"
openapi: "POST /v1beta/models/{model}:generateContent"
description: "Gera conteúdo usando o formato da API Google Gemini"
---

A LemonData suporta o formato nativo da API Google Gemini para modelos Gemini. Isso permite compatibilidade direta com os SDKs de IA do Google.

## Parâmetros de Caminho

<ParamField path="model" type="string" required>
  Nome do modelo (ex: `gemini-2.5-pro`, `gemini-2.5-flash`).
</ParamField>

## Parâmetros de Consulta

<ParamField query="key" type="string">
  Chave de API (alternativa à autenticação por cabeçalho).
</ParamField>

## Autenticação

Os endpoints do Gemini suportam múltiplos métodos de autenticação:
- `?key=YOUR_API_KEY` parâmetro de consulta
- `x-goog-api-key: YOUR_API_KEY` cabeçalho
- `Authorization: Bearer YOUR_API_KEY` cabeçalho

## Corpo da Requisição

<ParamField body="contents" type="array" required>
  Conteúdos da conversa.

  Cada objeto de conteúdo contém:
  - `role` (string): `user` ou `model`
  - `parts` (array): Partes do conteúdo (texto ou dados inline)
</ParamField>

<ParamField body="systemInstruction" type="object">
  Instrução de sistema para o modelo.
</ParamField>

<ParamField body="generationConfig" type="object">
  Configuração de geração:
  - `temperature` (number): Temperatura de amostragem
  - `topP` (number): Probabilidade de amostragem de núcleo
  - `topK` (integer): Amostragem Top-K
  - `maxOutputTokens` (integer): Máximo de tokens de saída
  - `stopSequences` (array): Sequências de parada
</ParamField>

<ParamField body="safetySettings" type="array">
  Configurações de filtro de segurança.
</ParamField>

## Resposta

<ResponseField name="candidates" type="array">
  Candidatos de conteúdo gerado.
</ResponseField>

<ResponseField name="usageMetadata" type="object">
  Informações de uso de tokens.
</ResponseField>
