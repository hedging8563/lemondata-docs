## التفويض

<ParamField header="Authorization" type="string" required>
  رمز Bearer للمصادقة على API. الصيغة: `Bearer sk-your-api-key`
</ParamField>

## جسم الطلب (Request Body)

<ParamField body="model" type="string" required>
  معرف (ID) النموذج المراد استخدامه. راجع [Models](https://lemondata.cc/ar/models) للخيارات المتاحة.
</ParamField>

<ParamField body="messages" type="array" required>
  قائمة من الرسائل التي تتكون منها المحادثة.

  يحتوي كل كائن رسالة على:
  - `role` (string): `system` أو `user` أو `assistant`
  - `content` (string | array): محتوى الرسالة
</ParamField>

<ParamField body="temperature" type="number" default="1">
  درجة حرارة أخذ العينات (Sampling temperature) بين 0 و 2. القيم الأعلى تجعل المخرجات أكثر عشوائية.
</ParamField>

<ParamField body="max_tokens" type="integer">
  الحد الأقصى لعدد الـ tokens المراد إنشاؤها.
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  إذا كانت true، فسيتم إرسال أجزاء الرسالة الجزئية كأحداث SSE.
</ParamField>

<ParamField body="stream_options" type="object">
  خيارات البث (Streaming). قم بتعيين `include_usage: true` لتلقي استخدام الـ tokens في أجزاء البث.
</ParamField>

<ParamField body="top_p" type="number" default="1">
  معلمة أخذ عينات النواة (Nucleus sampling). نوصي بتعديل هذا أو temperature، وليس كليهما.
</ParamField>

<ParamField body="frequency_penalty" type="number" default="0">
  رقم بين -2.0 و 2.0. القيم الإيجابية تعاقب الـ tokens المتكررة.
</ParamField>

<ParamField body="presence_penalty" type="number" default="0">
  رقم بين -2.0 و 2.0. القيم الإيجابية تعاقب الـ tokens الموجودة بالفعل في النص.
</ParamField>

<ParamField body="stop" type="string | array">
  ما يصل إلى 4 تسلسلات حيث سيتوقف الـ API عن إنشاء الـ tokens.
</ParamField>

<ParamField body="tools" type="array">
  قائمة بالأدوات التي قد يستدعيها النموذج (استدعاء الوظائف - function calling).
</ParamField>

<ParamField body="tool_choice" type="string | object">
  يتحكم في كيفية
