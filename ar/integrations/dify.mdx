## نظرة عامة

Dify هي منصة مفتوحة المصدر لتطوير تطبيقات LLM. توفر المنصة تنسيقاً مرئياً للمطالبات (prompts)، ومسارات RAG، وأطر عمل الوكلاء (agents)، وقدرات LLMOps. يمكن تكوين LemonData كمزود نماذج مخصص في Dify.

## الفوائد

- الوصول إلى أكثر من 300 نموذج ذكاء اصطناعي من خلال واجهة واحدة
- التبديل بين النماذج دون تغيير منطق التطبيق
- تحسين التكاليف عن طريق اختيار النموذج الأفضل لكل مهمة
- فوترة موحدة وتتبع الاستخدام

## المتطلبات المسبقة

- حساب LemonData مع وصول إلى API
- تثبيت Dify (سحابي أو استضافة ذاتية)

## خطوات التكوين

### الخطوة 1: الحصول على مفتاح API الخاص بك

1. قم بتسجيل الدخول إلى [LemonData Dashboard](https://lemondata.cc/dashboard)
2. انتقل إلى [API Keys](https://lemondata.cc/dashboard/api)
3. قم بإنشاء ونسخ مفتاح API الخاص بك (بتنسيق: `sk-...`)

### الخطوة 2: إضافة مزود نموذج مخصص

<Steps>
  <Step title="فتح الإعدادات">
    في Dify، انتقل إلى **Settings** ← **Model Provider**
  </Step>
  <Step title="إضافة مزود متوافق مع OpenAI">
    انقر فوق **Add Model Provider** وحدد **OpenAI-API-compatible**
  </Step>
  <Step title="تكوين المزود">
    أدخل الإعدادات التالية:

    | الحقل | القيمة |
    |-------|-------|
    | Provider Name | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="إضافة النماذج">
    أضف النماذج التي تريد استخدامها:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### الخطوة 3: اختبار الاتصال

1. اختر نموذجاً (مثل `gpt-4o-mini`)
2. أرسل رسالة اختبار
3. تحقق من تلقي استجابة

## الاستخدام في التطبيقات

### روبوت الدردشة (Chatbot)

1. أنشئ تطبيق Chatbot جديد
2. اختر LemonData كمزود للنماذج
3. اختر نموذجك المفضل
4. قم بتكوين مطالبات النظام (system prompt) والمعلمات

### الوكيل (Agent)

1. أنشئ تطبيق Agent
2. اختر نموذجاً قادراً (GPT-4o, Claude)
3. أضف الأدوات وقواعد المعرفة
4. تكوين سلوك الوكيل

### سير العمل (Workflow)

1. أنشئ سير عمل (Workflow)
2. أضف عقد LLM
3. اختر نماذج LemonData لكل عقدة
4. قم بتوصيل العقد وتكوين تدفق البيانات

## النماذج المتاحة

| الفئة | النماذج |
|----------|--------|
| الدردشة | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| التضمينات (Embeddings) | text-embedding-3-small, text-embedding-3-large |
| الرؤية (Vision) | GPT-4o (with images), Claude Sonnet (with images) |

## تكوين RAG

لتطبيقات RAG، قم بتكوين التضمينات (embeddings):

1. انتقل إلى **Settings** ← **Model Provider**
2. أضف نموذج التضمين: `text-embedding-3-small`
3. تعيينه كنموذج تضمين افتراضي في إعدادات قاعدة المعرفة
