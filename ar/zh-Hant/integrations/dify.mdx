## نظرة عامة

Dify هي منصة مفتوحة المصدر لتطوير تطبيقات LLM. توفر تنسيقاً مرئياً للـ Prompt، ومسارات RAG، وإطار عمل Agent، وقدرات LLMOps. يمكن تكوين LemonData كمزود نماذج مخصص في Dify.

## المزايا

- الوصول إلى أكثر من 300 نموذج AI عبر واجهة واحدة
- التبديل بين النماذج دون الحاجة لتغيير منطق التطبيق
- اختيار أفضل نموذج لكل مهمة لتحسين التكلفة
- فوترة موحدة وتتبع للاستخدام

## المتطلبات المسبقة

- حساب LemonData مع صلاحية الوصول إلى API
- تثبيت Dify (الإصدار السحابي أو النشر الخاص)

## خطوات التكوين

### الخطوة 1: الحصول على API Key الخاص بك

1. قم بتسجيل الدخول إلى [LemonData Dashboard](https://lemondata.cc/dashboard)
2. انتقل إلى [API Keys](https://lemondata.cc/dashboard/api)
3. قم بإنشاء ونسخ API Key الخاص بك (بصيغة: `sk-...`)

### الخطوة 2: إضافة مزود نماذج مخصص

<Steps>
  <Step title="فتح الإعدادات">
    في Dify، انتقل إلى **Settings** ← **Model Provider**
  </Step>
  <Step title="إضافة مزود متوافق مع OpenAI">
    انقر فوق **Add Model Provider** واختر **OpenAI-API-compatible**
  </Step>
  <Step title="تكوين المزود">
    أدخل الإعدادات التالية:

    | الحقل | القيمة |
    |-------|-------|
    | اسم المزود | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="إضافة النماذج">
    أضف النماذج التي ترغب في استخدامها:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### الخطوة 3: اختبار الاتصال

1. اختر نموذجاً (مثل `gpt-4o-mini`)
2. أرسل رسالة اختبار
3. تأكد من استلام الرد

## الاستخدام في التطبيقات

### روبوت الدردشة (Chatbot)

1. أنشئ تطبيق Chatbot جديد
2. اختر LemonData كمزود للنماذج
3. اختر النموذج المفضل لديك
4. قم بتكوين System Prompt والمعلمات

### الوكيل (Agent)

1. أنشئ تطبيق Agent
2. اختر نموذجاً قوياً (GPT-4o, Claude)
3. أضف الأدوات وقواعد المعرفة
4. قم بتكوين سلوك Agent

### سير العمل (Workflow)

1. أنشئ سير عمل
2. أضف عقدة LLM
3. اختر نموذج LemonData لكل عقدة
4. قم بتوصيل العقد وتكوين تدفق البيانات

## النماذج المتاحة

| الفئة | النماذج |
|----------|--------|
| الدردشة (Chat) | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| التضمينات (Embeddings) | text-embedding-3-small, text-embedding-3-large |
| الرؤية (Vision) | GPT-4o (يدعم الصور), Claude Sonnet (يدعم الصور) |

## تكوين RAG

بالنسبة لتطبيقات RAG، يرجى تكوين نماذج التضمين:

1. انتقل إلى **Settings** ← **Model Provider**
2. أضف نموذج التضمين: `text-embedding-3-small`
3. قم بتعيينه كنموذج تضمين افتراضي في إعدادات قاعدة المعرفة
