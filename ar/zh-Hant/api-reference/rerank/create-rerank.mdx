---

استخدم نماذج التشابه الدلالي لإعادة ترتيب المستندات. مناسب لتحسين نتائج البحث وتطبيقات RAG.

## جسم الطلب

<ParamField body="model" type="string" required>
  معرف نموذج إعادة الترتيب المراد استخدامه (مثال: `BAAI/bge-reranker-v2-m3`، `qwen3-rerank`).
</ParamField>

<ParamField body="query" type="string" required>
  جملة الاستعلام المستخدمة لتصنيف المستندات.
</ParamField>

<ParamField body="documents" type="array" required>
  قائمة المستندات المراد إعادة ترتيبها (سلاسل نصية).
</ParamField>

<ParamField body="top_n" type="integer">
  عدد النتائج الأولى التي سيتم إرجاعها. الافتراضي هو جميع المستندات.
</ParamField>

<ParamField body="return_documents" type="boolean" default="false">
  ما إذا كان سيتم تضمين نص المستند الأصلي في الاستجابة.
</ParamField>

## الاستجابة

<ResponseField name="results" type="array">
  قائمة المستندات المرتبة مع الدرجات.

  يحتوي كل نتيجة على:
  - `index` (integer): فهرس المستند الأصلي
  - `relevance_score` (number): درجة الصلة (0-1)
  - `document` (string): النص الأصلي (إذا كان `return_documents=true`)
</ResponseField>

<ResponseField name="model" type="string">
  النموذج المستخدم لإعادة الترتيب.
</ResponseField>

<ResponseField name="usage" type="object">
  إحصائيات استخدام Token.
</ResponseField>
