## جسم الطلب (Request Body)

<ParamField body="model" type="string" required>
  معرف النموذج المراد استخدامه. يرجى مراجعة [Models](https://lemondata.cc/zh-TW/models) للاطلاع على الخيارات المتاحة.
</ParamField>

<ParamField body="messages" type="array" required>
  قائمة الرسائل التي تشكل المحادثة.

  يحتوي كل كائن رسالة على:
  - `role` (string): `system` أو `user` أو `assistant`
  - `content` (string | array): محتوى الرسالة
</ParamField>

<ParamField body="temperature" type="number" default="1">
  درجة حرارة أخذ العينات بين 0 و 2. القيم الأعلى تجعل المخرجات أكثر عشوائية.
</ParamField>

<ParamField body="max_tokens" type="integer">
  الحد الأقصى لعدد الـ tokens المراد إنشاؤها.
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  إذا كانت true، فسيتم إرسال أجزاء من الرسائل بشكل تدريجي كأحداث SSE.
</ParamField>

<ParamField body="stream_options" type="object">
  خيارات البث. قم بتعيين `include_usage: true` لتلقي تفاصيل استخدام الـ token في أجزاء البث.
</ParamField>

<ParamField body="top_p" type="number" default="1">
  معلمة أخذ العينات النووية (Nucleus sampling). نوصي بتعديل هذه المعلمة أو temperature، وليس كليهما في وقت واحد.
</ParamField>

<ParamField body="frequency_penalty" type="number" default="0">
  قيمة بين -2.0 و 2.0. القيم الإيجابية تعاقب الـ tokens المتكررة.
</ParamField>

<ParamField body="presence_penalty" type="number" default="0">
  قيمة بين -2.0 و 2.0. القيم الإيجابية تعاقب الـ tokens التي ظهرت بالفعل في النص.
</ParamField>

<ParamField body="stop" type="string | array">
  ما يصل إلى 4 تسلسلات، حيث سيتوقف الـ API عن إنشاء الـ tokens عند هذه التسلسلات.
</ParamField>

<ParamField body="tools" type="array">
  قائمة بالأدوات التي قد يستدعيها النموذج (function calling).
</ParamField>

<ParamField body="tool_choice" type="string | object">
  يتحكم في كيفية استخدام النموذج للأدوات. الخيارات: `auto` أو `none` أو `required` أو كائن أداة محدد.
</ParamField>

<ParamField body="parallel_tool_calls" type="boolean" default="true">
  ما إذا كان سيتم تمكين استدعاءات الوظائف المتوازية. اضبطها على false لاستدعاء الوظائف بالتسلسل.
</ParamField>

<ParamField body="max_completion_tokens" type="integer">
  الحد الأقصى لعدد tokens الإكمال. بديل لـ `max_tokens` ويُنصح به للنماذج الأحدث مثل o1/o3.
</ParamField>

<ParamField body="reasoning_effort" type="string">
  جهد الاستنتاج لنماذج o1/o3. الخيارات: `low` أو `medium` أو `high`.
</ParamField>

<ParamField body="seed" type="integer">
  بذرة عشوائية لأخذ العينات الحتمي.
</ParamField>

<ParamField body="n" type="integer" default="1">
  عدد الإكمالات المراد إنشاؤها (1-128).
</ParamField>

<ParamField body="logprobs" type="boolean">
  ما إذا كان سيتم إرجاع الاحتمالات اللوغاريتمية (log probabilities).
</ParamField>

<ParamField body="top_logprobs" type="integer">
  عدد الاحتمالات اللوغاريتمية الأعلى التي سيتم إرجاعها (0-20). يتطلب تعيين `logprobs: true`.
</ParamField>

<ParamField body="top_k" type="integer">
  معلمة أخذ العينات Top-K (تنطبق على نماذج Anthropic/Gemini).
</ParamField>

<ParamField body="response_format" type="object">
  مواصفات تنسيق الاستجابة. استخدم `{"type": "json_object"}` لتمكين وضع JSON، أو `{"type": "json_schema", "json_schema": {...}}` للمخرجات المهيكلة.
</ParamField>

<ParamField body="logit_bias" type="object">
  تعديل احتمالية ظهور tokens معينة. يربط معرف الـ token (بصيغة سلسلة نصية) بقيمة انحياز بين -100 و 100.
</ParamField>

<ParamField body="user" type="string">
  معرف فريد يمثل المستخدم النهائي الخاص بك، يستخدم لمراقبة سوء الاستخدام.
</ParamField>

<ParamField body="cache_control" type="object">
  خيارات التحكم في التخزين المؤقت لـ LemonData.

  - `type` (string): استراتيجية التخزين المؤقت - `default` أو `no_cache` أو `no_store` أو `response_only` أو `semantic_only`
  - `max_age` (integer): وقت بقاء التخزين المؤقت (TTL) بالثواني (بحد أقصى 86400)
</ParamField>

## الاستجابة (Response)

<ResponseField name="id" type="string">
  المعرف الفريد للإكمال.
</ResponseField>

<ResponseField name="object" type="string">
  دائماً `chat.completion`.
</ResponseField>

<ResponseField name="created" type="integer">
  طابع Unix الزمني لوقت إنشاء الإكمال.
</ResponseField>

<ResponseField name="model" type="string">
  النموذج المستخدم للإكمال.
</ResponseField>

<ResponseField name="choices" type="array">
  قائمة خيارات الإكمال.

  يحتوي كل خيار على:
  - `index` (integer): فهرس الخيار
  - `message` (object): الرسالة التي تم إنشاؤها
  - `finish_reason` (string): سبب توقف النموذج (`stop` أو `length` أو `tool_calls`)
</ResponseField>

<ResponseField name="usage" type="object">
  إحصائيات استخدام الـ Token.

  - `prompt_tokens` (integer): عدد الـ tokens في المطالبة
  - `completion_tokens` (integer): عدد الـ tokens في الإكمال
  - `total_tokens` (integer): إجمالي عدد الـ tokens المستخدمة
</ResponseField>

```python Python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.lemondata.cc/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"}
    ],
    temperature=0.7,
    max_tokens=1000
)

print(response.choices[0].message.content)
```

```go Go
package main

import (
    "context"
    "fmt"
    "github.com/sashabaranov/go-openai"
)

func main() {
    config := openai.DefaultConfig("sk-your-api-key")
    config.BaseURL = "https://api.lemondata.cc/v1"
    client := openai.NewClientWithConfig(config)

    resp, err := client.CreateChatCompletion(
        context.Background(),
        openai.ChatCompletionRequest{
            Model: "gpt-4o",
            Messages: []openai.ChatCompletionMessage{
                {Role: openai.ChatMessageRoleSystem, Content: "You are a helpful assistant."},
                {Role: openai.ChatMessageRoleUser, Content: "Hello!"},
            },
            Temperature: 0.7,
            MaxTokens:   1000,
        },
    )
    if err != nil {
        panic(err)
    }
    fmt.Println(resp.Choices[0].Message.Content)
}
```
