## مفاتيح API

تتطلب جميع طلبات API المصادقة باستخدام مفتاح API. يرجى تضمين مفتاحك في ترويسة `Authorization`:

```bash
Authorization: Bearer sk-your-api-key
```

## الحصول على مفتاح API الخاص بك

1. قم بتسجيل الدخول إلى [LemonData Dashboard](https://lemondata.cc/dashboard)
2. انتقل إلى قسم **API Keys**
3. انقر فوق **Create New Key**
4. قم بتسمية مفتاحك باسم وصفي
5. انسخ المفتاح فوراً — سيتم عرضه مرة واحدة فقط

<Warning>
  **أفضل ممارسات الأمان:**
  - لا تكشف أبداً عن مفاتيح API في كود جانب العميل (client-side code)
  - لا تقم بإرسال المفاتيح إلى أنظمة التحكم في الإصدار (version control systems)
  - استخدم متغيرات البيئة لتخزين المفاتيح
  - قم بتدوير المفاتيح بشكل دوري
  - احذف المفاتيح غير المستخدمة
</Warning>

## استخدام مفاتيح API

<CodeGroup>

```bash cURL
curl https://api.lemondata.cc/v1/chat/completions \
  -H "Authorization: Bearer $LEMONDATA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello!"}]}'
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("LEMONDATA_API_KEY"),
    base_url="https://api.lemondata.cc/v1"
)
```

```javascript JavaScript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: process.env.LEMONDATA_API_KEY,
  baseURL: 'https://api.lemondata.cc/v1'
});
```

```go Go
package main

import "github.com/sashabaranov/go-openai"

func main() {
    config := openai.DefaultConfig(os.Getenv("LEMONDATA_API_KEY"))
    config.BaseURL = "https://api.lemondata.cc/v1"
    client := openai.NewClientWithConfig(config)
}
```

```php PHP
<?php
$apiKey = getenv('LEMONDATA_API_KEY');

$ch = curl_init('https://api.lemondata.cc/v1/chat/completions');
curl_setopt_array($ch, [
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer ' . $apiKey,
        'Content-Type: application/json'
    ]
]);
```

</CodeGroup>

## ميزات مفاتيح API

### قيود الاستخدام

يمكنك تعيين قيود الاستخدام لكل مفتاح API للتحكم في التكاليف:

| الإعداد | الوصف |
|---------|-------------|
| **بلا قيود** | سيستخدم المفتاح رصيد حسابك دون قيود |
| **حد ثابت** | سيتوقف المفتاح عن العمل بعد الوصول إلى المبلغ المحدد |

### بادئة المفتاح

تبدأ جميع مفاتيح LemonData API بالبادئة `sk-`. تنسيق المفتاح هو:

```
sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## التوافق مع Anthropic API

بالنسبة لنقطة النهاية `/v1/messages`، يمكنك استخدام ترويسة `x-api-key` (متوافقة مع Anthropic SDK):

```bash
curl https://api.lemondata.cc/v1/messages \
  -H "x-api-key: sk-your-api-key" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-5",
    "max_tokens": 1024,
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

<Note>
  تدعم ترويسة `x-api-key` فقط نقاط النهاية `/v1/messages` و `/v1/messages/count_tokens`. تتطلب نقاط النهاية الأخرى ترويسة `Authorization: Bearer`.
</Note>

## استجابات الخطأ

| رمز الحالة | النوع | الرمز | الوصف |
|-------------|------|------|-------------|
| 401 | `invalid_request_error` | `invalid_api_key` | مفتاح API مفقود أو غير صالح |
| 401 | `invalid_request_error` | `expired_api_key` | تم إلغاء مفتاح API |
| 402 | `insufficient_quota` | `insufficient_quota` | رصيد الحساب غير كافٍ |
| 402 | `insufficient_quota` | `quota_exceeded` | تم الوصول إلى حد استخدام مفتاح API |

مثال على استجابة الخطأ:

```json
{
  "error": {
    "message": "Invalid API key provided",
    "type": "invalid_api_key",
    "code": "invalid_api_key"
  }
}
```
