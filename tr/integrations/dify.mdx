## Genel Bakış

Dify, açık kaynaklı bir LLM uygulama geliştirme platformudur. Görsel prompt orkestrasyonu, RAG boru hatları (pipelines), ajan çerçeveleri ve LLMOps yetenekleri sunar. LemonData, Dify'da özel bir model sağlayıcısı olarak yapılandırılabilir.

## Avantajlar

- Tek bir arayüz üzerinden 300'den fazla yapay zeka modeline erişin
- Uygulama mantığını değiştirmeden modeller arasında geçiş yapın
- Her görev için en iyi modeli seçerek maliyetleri optimize edin
- Birleştirilmiş faturalandırma ve kullanım takibi

## Ön Koşullar

- API erişimi olan LemonData hesabı
- Dify kurulumu (bulut veya self-hosted)

## Yapılandırma Adımları

### Adım 1: API Anahtarınızı Alın

1. [LemonData Dashboard](https://lemondata.cc/dashboard) adresine giriş yapın
2. [API Keys](https://lemondata.cc/dashboard/api) bölümüne gidin
3. API anahtarınızı oluşturun ve kopyalayın (format: `sk-...`)

### Adım 2: Özel Model Sağlayıcısı Ekleme

<Steps>
  <Step title="Ayarları Açın">
    Dify'da **Settings** → **Model Provider** bölümüne gidin
  </Step>
  <Step title="OpenAI Uyumlu Sağlayıcı Ekle">
    **Add Model Provider**'a tıklayın ve **OpenAI-API-compatible** seçeneğini seçin
  </Step>
  <Step title="Sağlayıcıyı Yapılandır">
    Aşağıdaki ayarları girin:

    | Alan | Değer |
    |-------|-------|
    | Provider Name | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="Modelleri Ekle">
    Kullanmak istediğiniz modelleri ekleyin:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### Adım 3: Bağlantıyı Test Edin

1. Bir model seçin (örneğin, `gpt-4o-mini`)
2. Bir test mesajı gönderin
3. Yanıt aldığınızı doğrulayın

## Uygulamalarda Kullanım

### Chatbot

1. Yeni bir Chatbot uygulaması oluşturun
2. Model sağlayıcısı olarak LemonData'yı seçin
3. Tercih ettiğiniz modeli seçin
4. Sistem prompt'unu ve parametreleri yapılandırın

### Agent

1. Bir Agent uygulaması oluşturun
2. Yetenekli bir model seçin (GPT-4o, Claude)
3. Araçlar ve bilgi tabanları (knowledge bases) ekleyin
4. Agent davranışını yapılandırın

### Workflow

1. Bir Workflow oluşturun
2. LLM düğümleri (nodes) ekleyin
3. Her düğüm için LemonData modellerini seçin
4. Düğümleri bağlayın ve veri akışını yapılandırın

## Mevcut Modeller

| Kategori | Modeller |
|----------|--------|
| Sohbet | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| Embeddings | text-embedding-3-small, text-embedding-3-large |
| Görüntü İşleme (Vision) | GPT-4o (görsellerle), Claude Sonnet (görsellerle) |

## RAG Yapılandırması

RAG uygulamaları için embedding'leri yapılandırın:

1. **Settings** → **Model Provider** bölümüne gidin
2. Embedding modeli ekleyin: `text-embedding-3-small`
3. Bilgi tabanı ayarlarında varsayılan embedding modeli olarak ayarlayın

## Sorun Giderme

<AccordionGroup>
  <Accordion title="Bağlantı Hatası">
    - API Base URL'nin tam olarak `https://api.lemondata.cc/v1` olduğunu doğrulayın
    - Sondaki eğik çizgilere (slash) dikkat edin
    - Ağ bağlantısını doğrulayın
  </Accordion>

  <Accordion title="401 Yetkisiz Erişim (Unauthorized)">
    - API anahtarını tekrar kontrol edin
    - Anahtarın LemonData panelinde aktif olduğundan emin olun
  </Accordion>

  <Accordion title="Model Bulunamadı">
    - Model adının tam olarak eşleştiğini doğrulayın
    - Modelin kullanılabilirliğini [lemondata.cc/en/models](https://lemondata.cc/tr/models) adresinden kontrol edin
  </Accordion>
</AccordionGroup>
