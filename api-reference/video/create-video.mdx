## Overview

Video generation is asynchronous. You submit a request and receive a task ID, then poll for the result.

## Request Body

<ParamField body="model" type="string" default="sora-2">
  Video model ID. See the [Video Generation Guide](/guides/video-generation) for a full list of 80+ models across 11 providers.
</ParamField>

<ParamField body="prompt" type="string" required>
  Text description of the video to generate.
</ParamField>

<ParamField body="image_url" type="string">
  URL of starting image for image-to-video (I2V) generation.
</ParamField>

<ParamField body="image" type="string">
  Base64-encoded image with data URL prefix (e.g., `data:image/jpeg;base64,...`). Alternative to `image_url` for I2V generation.
</ParamField>

<ParamField body="duration" type="integer">
  Video duration in seconds (model-dependent).
</ParamField>

<ParamField body="aspect_ratio" type="string">
  Aspect ratio (e.g., `16:9`, `9:16`, `1:1`).
</ParamField>

<ParamField body="resolution" type="string">
  Video resolution (e.g., `1080p`, `720p`).
</ParamField>

<ParamField body="fps" type="integer">
  Frames per second (1-120).
</ParamField>

<ParamField body="negative_prompt" type="string">
  What to avoid in the video generation.
</ParamField>

<ParamField body="seed" type="integer">
  Random seed for reproducible generation.
</ParamField>

<ParamField body="cfg_scale" type="number">
  Classifier-free guidance scale (0-20).
</ParamField>

<ParamField body="motion_strength" type="number">
  Motion intensity (0-1).
</ParamField>

<ParamField body="start_image" type="string">
  URL of starting frame image.
</ParamField>

<ParamField body="end_image" type="string">
  URL of ending frame image.
</ParamField>

<ParamField body="user" type="string">
  A unique identifier for the end-user.
</ParamField>

## Response

<ResponseField name="task_id" type="string">
  Unique task identifier for polling.
</ResponseField>

<ResponseField name="status" type="string">
  Initial status: `pending`.
</ResponseField>

<ResponseField name="model" type="string">
  Model used.
</ResponseField>

```python Python
import requests

        "prompt": "A cat walking through a garden, cinematic lighting",
        "duration": 5
```

```go Go
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
)

func main() {
    payload := map[string]interface{}{
        "model":    "kling-v2.6-pro",
        "prompt":   "A cat walking through a garden, cinematic lighting",
        "duration": 5,
    }
    body, _ := json.Marshal(payload)

    req, _ := http.NewRequest("POST", "https://api.lemondata.cc/v1/videos/generations", bytes.NewBuffer(body))
    req.Header.Set("Authorization", "Bearer sk-your-api-key")
    req.Header.Set("Content-Type", "application/json")

    client := &http.Client{}
    resp, _ := client.Do(req)
    defer resp.Body.Close()

    var result map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&result)
    fmt.Printf("Task ID: %s\n", result["task_id"])
}
```

## Image to Video

```python
response = requests.post(
    "https://api.lemondata.cc/v1/videos/generations",
    headers={"Authorization": "Bearer sk-your-api-key"},
    json={
        "model": "wan2.6-i2v",
        "prompt": "The person starts walking forward",
        "image_url": "https://example.com/image.jpg"
    }
)
```

## Keyframe Control

Use `start_image` and `end_image` to control the first and last frames (supported by Seedance 2.0):

```python
response = requests.post(
    "https://api.lemondata.cc/v1/videos/generations",
    headers={"Authorization": "Bearer sk-your-api-key"},
    json={
        "model": "seedance-2-0",
        "prompt": "Smooth transition from day to night",
        "start_image": "https://example.com/day.jpg",
        "end_image": "https://example.com/night.jpg",
        "duration": 5
    }
)
```
