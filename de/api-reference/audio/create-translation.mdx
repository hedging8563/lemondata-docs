## Übersicht

Übersetzt Audio in jeder unterstützten Sprache in englischen Text. Im Gegensatz zur Transkription gibt dieser Endpunkt unabhängig von der Eingabesprache immer englischen Text aus.

## Request Body

<ParamField body="file" type="file" required>
  Die zu übersetzende Audiodatei. Unterstützte Formate: `flac`, `mp3`, `mp4`, `mpeg`, `mpga`, `m4a`, `ogg`, `wav`, `webm`. Die maximale Dateigröße beträgt 25 MB.
</ParamField>

<ParamField body="model" type="string" default="whisper-1">
  Das zu verwendende Modell. Derzeit wird nur `whisper-1` unterstützt.
</ParamField>

<ParamField body="prompt" type="string">
  Ein optionaler Text, um den Stil des Modells zu steuern oder ein vorheriges Segment fortzusetzen. Sollte auf Englisch sein.
</ParamField>

<ParamField body="response_format" type="string" default="json">
  Das Format der Ausgabe. Optionen: `json`, `text`, `srt`, `verbose_json`, `vtt`.
</ParamField>

<ParamField body="temperature" type="number">
  Die Sampling-Temperatur, zwischen 0 und 1. Höhere Werte wie 0,8 erzeugen eine zufälligere Ausgabe, während niedrigere Werte wie 0,2 die Ausgabe fokussierter und deterministischer machen.
</ParamField>

## Antwort

<ResponseField name="text" type="string">
  Der übersetzte Text auf Englisch.
</ResponseField>

Für das `verbose_json`-Format enthält die Antwort zusätzlich:

<ResponseField name="language" type="string">
  Die erkannte Sprache des Eingabe-Audios.
</ResponseField>

<ResponseField name="duration" type="number">
  Die Dauer des Eingabe-Audios in Sekunden.
</ResponseField>

<ResponseField name="segments" type="array">
  Segmente des übersetzten Textes mit Zeitstempeln.
</ResponseField>

<RequestExample>

```bash cURL
  -H "Authorization: Bearer sk-your-api-key" \

```
