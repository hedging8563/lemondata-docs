Verbinden Sie DingTalk (钉钉) mit Ihrer Claw-Instanz im Stream-Modus – eine WebSocket-basierte Verbindung, die keine öffentliche URL erfordert.

## Voraussetzungen

- Ein [DingTalk](https://www.dingtalk.com/)-Konto mit Administratorrechten für Unternehmen
- Eine Claw-Instanz ([hier erstellen](https://lemondata.cc/dashboard/claw))

## Einrichtungsanleitung

<Steps>
  <Step title="Eine DingTalk-App erstellen">
    1. Gehen Sie zur [DingTalk Open Platform](https://open-dev.dingtalk.com/)
    2. Klicken Sie auf **Application Development** → **Enterprise Internal Development**
    3. Klicken Sie auf **Create Application**
    4. Geben Sie einen App-Namen und eine Beschreibung ein
    5. Klicken Sie auf **Confirm**
  </Step>
  <Step title="Robot-Funktion aktivieren">
    1. Gehen Sie in Ihrer App zu **App Features** → **Robot**
    2. Aktivieren Sie die Robot-Konfiguration
    3. Wählen Sie den **Stream Mode** (empfohlen – keine öffentliche URL erforderlich)
    4. Klicken Sie auf **Publish**

    <Tip>
    Der Stream-Modus verwendet WebSocket, sodass Ihre Claw-Instanz keine öffentlich zugängliche Webhook-URL benötigt. Dies ist die einfachste Einrichtung.
    </Tip>
  </Step>
  <Step title="Anmeldedaten abrufen">
    1. Gehen Sie zu **Basic Information** → **Credentials**
    2. Kopieren Sie den **AppKey** und das **AppSecret**
  </Step>
  <Step title="Die App veröffentlichen">
    1. Gehen Sie zu **Version Management**
    2. Erstellen Sie eine neue Version (mindestens eine Testversion)
    3. Legen Sie den Sichtbarkeitsbereich fest (welche Benutzer/Abteilungen den Bot sehen können)
    4. Einreichen und veröffentlichen
  </Step>
  <Step title="Im LemonData Dashboard konfigurieren">
    1. Gehen Sie zu Ihrem [Claw Dashboard](https://lemondata.cc/dashboard/claw)
    2. Klicken Sie auf das **Einstellungen-Symbol** Ihrer Instanz
    3. Öffnen Sie den Tab **IM Integrations**
    4. Erweitern Sie den Bereich **DingTalk**
    5. Fügen Sie Ihren **AppKey** und Ihr **AppSecret** ein
    6. Klicken Sie auf **Save**
  </Step>
  <Step title="Chatten starten">
    1. Suchen Sie in DingTalk nach Ihrem Bot anhand seines Namens
    2. Senden Sie ihm eine Nachricht
    3. Der erste Benutzer wird **automatisch als Besitzer gekoppelt** – kein Code erforderlich!
    4. Beginnen Sie mit der KI zu chatten!
  </Step>
</Steps>

## Gruppeneinstellungen

Konfigurieren Sie das Verhalten von Gruppenchats im Tab **IM Integrations** Ihres Dashboards:

| Einstellung | Optionen | Beschreibung |
|---------|---------|-------------|
| **Group Policy** | Open (Standard) / Allowlist / Disabled | Steuert, wer den Bot in Gruppen auslösen kann |
| **Group Allowed Users** | Liste von Benutzer-IDs | Nur diese Benutzer können den Bot auslösen (Allowlist-Modus) |

<Note>
DingTalk **erfordert immer eine @Erwähnung**, um den Bot in Gruppen auszulösen – dies ist eine Einschränkung der Plattform und kann nicht konfiguriert werden. Die Filterung nach Gruppen-IDs ist für DingTalk nicht verfügbar; verwenden Sie die Einstellung **Group Policy**, um den Zugriff zu steuern.
</Note>

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="Bot antwortet nicht">
    - Überprüfen Sie, ob **AppKey** und **AppSecret** korrekt sind
    - Stellen Sie sicher, dass die App veröffentlicht ist (mindestens Testversion)
    - Vergewissern Sie sich, dass der **Stream Mode** in der Robot-Konfiguration ausgewählt ist
    - Überprüfen Sie, ob der Bot für Ihr Benutzerkonto sichtbar ist
  </Accordion>
  <Accordion title="Verbindung bricht häufig ab">
    - Der Stream-Modus hält eine dauerhafte WebSocket-Verbindung aufrecht
    - Kurze Verbindungsabbrüche sind normal – der Bot wird sich automatisch neu verbinden
    - Falls das Problem bestehen bleibt, versuchen Sie, die Instanz über das Dashboard neu zu starten
  </Accordion>
</AccordionGroup>
