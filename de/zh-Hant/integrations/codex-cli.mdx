## Übersicht

OpenAI Codex ist ein Open-Source-Kommandozeilenwerkzeug (CLI), das als leichtgewichtiger Coding-Agent fungiert und Code im Terminal lesen, ändern und ausführen kann. Es basiert auf GPT-Modellen und ist für die Codegenerierung optimiert.

## Systemanforderungen

- **Betriebssystem**: macOS, Linux (offizielle Unterstützung), Windows über WSL
- **Node.js**: Version 18+
- **npm**: Version 10.x.x oder höher

## Installation

```bash
sudo npm install -g @openai/codex@latest
```

Installation überprüfen:

```bash
codex --version
```

## Konfiguration

### Schritt 1: API-Key festlegen

**Temporär (aktuelle Sitzung):**

```bash
export OPENAI_API_KEY="sk-your-lemondata-key"
```

**Permanente Konfiguration:**

Hinzufügen zu `~/.bashrc`, `~/.zshrc` oder `~/.bash_profile`:

```bash
export OPENAI_API_KEY="sk-your-lemondata-key"
```

Anschließend neu laden:

```bash
source ~/.zshrc  # oder source ~/.bashrc
```

### Schritt 2: config.toml konfigurieren

Bearbeiten Sie `~/.codex/config.toml`:

```toml
model = "gpt-4o"
model_provider = "lemondata"

[model_providers.lemondata]
name = "LemonData"
base_url = "https://api.lemondata.cc/v1"
env_key = "OPENAI_API_KEY"
wire_api = "chat"
```

<Note>
Falls die Konfigurationsdatei nicht existiert, führen Sie `codex` einmal aus, um die Datei zu generieren, und bearbeiten Sie sie anschließend.
</Note>

## Grundlegende Verwendung

**Interaktiven Modus starten:**

```bash
codex
```

**Direkter Befehl:**

```bash
codex "Fix the bug in main.py line 42"
```

**Modell angeben:**

```bash
codex -m gpt-4o "Build a REST API server"
```

## Verfügbare Modelle

| Modell | Anwendungsfälle |
|-------|----------|
| `gpt-4o` | Komplexe Coding-Aufgaben, Architekturdesign |
| `gpt-4o-mini` | Schnelle Fixes, einfache Aufgaben |
| `claude-sonnet-4-5` | Code-Review, Dokumentationserstellung |
| `deepseek-r1` | Algorithmus-Design, Reasoning |

## Interaktive Befehle

| Befehl | Beschreibung |
|---------|-------------|
| `/help` | Hilfe anzeigen |
| `/exit` oder `Ctrl+C` | Beenden |
| `/clear` | Konversation löschen |
| `/config` | Konfiguration anzeigen |
| `/model <name>` | Modell wechseln |
| `/tokens` | Token-Verbrauch anzeigen |

## Konfiguration überprüfen

```bash
# Umgebungsvariablen prüfen
echo $OPENAI_API_KEY

# API-Verbindung testen
codex "Hello, Codex!"

# Konfiguration anzeigen
cat ~/.codex/config.toml
```

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="Verbindungsfehler">
    - Überprüfen Sie, ob `base_url` in der `config.toml` exakt `https://api.lemondata.cc/v1` entspricht
    - Netzwerkverbindung prüfen
    - Sicherstellen, dass keine Proxy-Server stören
  </Accordion>

  <Accordion title="Authentifizierung fehlgeschlagen">
    - Überprüfen Sie, ob die Umgebungsvariable `OPENAI_API_KEY` gesetzt ist
    - Sicherstellen, dass der Key mit `sk-` beginnt
    - Sicherstellen, dass der Key im LemonData-Dashboard aktiviert ist
  </Accordion>

  <Accordion title="Modell nicht gefunden">
    - Überprüfen Sie, ob der Modellname exakt übereinstimmt
    - Verfügbarkeit des Modells unter [lemondata.cc/en/models](https://lemondata.cc/zh-TW/models) prüfen
  </Accordion>
</AccordionGroup>
