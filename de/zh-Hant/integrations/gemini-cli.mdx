## Übersicht

Das Google Gemini CLI ist ein Befehlszeilentool zur Interaktion mit Gemini-Modellen. LemonData bietet einen kompatiblen Endpunkt, der es Ihnen ermöglicht, das Gemini CLI zu verwenden und auf über 300 Modelle zuzugreifen.

## Systemanforderungen

- **Node.js**: Version 20.0+
- **Betriebssystem**: Windows 10/11, macOS 10.15+, Ubuntu 20.04+, oder Debian 10+

## Installation

```bash
npm install -g @google/gemini-cli
```

Installation überprüfen:

```bash
gemini --version
```

## Konfiguration

### Schritt 1: Erhalten Sie Ihren API Key

1. Melden Sie sich im [LemonData Dashboard](https://lemondata.cc/dashboard) an
2. Navigieren Sie zu [API Keys](https://lemondata.cc/dashboard/api)
3. Erstellen und kopieren Sie Ihren API Key (Format: `sk-...`)

### Schritt 2: Umgebungsvariablen festlegen

**Temporär (aktuelle Sitzung):**

```bash
export GEMINI_API_KEY="sk-your-lemondata-key"
export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"
```

**Permanente Konfiguration:**

Fügen Sie dies zu Ihrer Shell-Konfigurationsdatei hinzu:

<Tabs>
  <Tab title="Bash">
    ```bash
    echo 'export GEMINI_API_KEY="sk-your-lemondata-key"' >> ~/.bashrc
    echo 'export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"' >> ~/.bashrc
    source ~/.bashrc
    ```
  </Tab>
  <Tab title="Zsh">
    ```bash
    echo 'export GEMINI_API_KEY="sk-your-lemondata-key"' >> ~/.zshrc
    echo 'export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"' >> ~/.zshrc
    source ~/.zshrc
    ```
  </Tab>
  <Tab title="Fish">
    ```bash
    set -Ux GEMINI_API_KEY "sk-your-lemondata-key"
    set -Ux GOOGLE_GEMINI_BASE_URL "https://api.lemondata.cc"
    ```
  </Tab>
</Tabs>

<Warning>
Bitte starten Sie Ihr Terminal nach der Konfiguration neu, damit die Änderungen wirksam werden.
</Warning>

<Note>
**Bekannte Einschränkungen**: In einigen Fällen folgt das Gemini CLI möglicherweise nicht der `GOOGLE_GEMINI_BASE_URL`, wenn eine zwischengespeicherte Google-Sitzung vorhanden ist. Wenn Verbindungsprobleme auftreten, versuchen Sie, den CLI-Cache zu leeren oder eine neue Terminal-Sitzung zu starten.
</Note>

## Grundlegende Verwendung

Starten Sie das Gemini CLI aus Ihrem Projektverzeichnis:

```bash
cd your-project
gemini
```

Beim ersten Ausführen müssen Sie:
1. Ein Thema auswählen
2. Sicherheitshinweise bestätigen
3. Dem Arbeitsverzeichnis vertrauen

## Fehlerbehebung

<AccordionGroup>
  <Accordion title="Verbindungsfehler">
    - Überprüfen Sie, ob `GOOGLE_GEMINI_BASE_URL` auf `https://api.lemondata.cc` eingestellt ist
    - Hinweis: Gemini-Endpunkte benötigen kein `/v1`-Suffix
    - Überprüfen Sie die Netzwerkverbindung
  </Accordion>

  <Accordion title="Authentifizierung fehlgeschlagen">
    - Überprüfen Sie, ob die Umgebungsvariable `GEMINI_API_KEY` gesetzt ist
    - Prüfen Sie, ob der Key mit `sk-` beginnt
    - Stellen Sie sicher, dass der Key im LemonData Dashboard aktiviert ist
  </Accordion>

  <Accordion title="Modell nicht verfügbar">
    - Prüfen Sie die Verfügbarkeit von Gemini-Modellen unter [lemondata.cc/en/models](https://lemondata.cc/zh-TW/models)
    - Versuchen Sie verschiedene Gemini-Modellvarianten
  </Accordion>
</AccordionGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="In Projektverzeichnissen verwenden">
    Führen Sie das Gemini CLI immer aus dem Projektstammverzeichnis aus, um einen besseren Kontext zu ermöglichen.
  </Accordion>

  <Accordion title="Verzeichnissen vorsichtig vertrauen">
    Vertrauen Sie nur Verzeichnissen, die Ihnen gehören. Das Gemini CLI kann Dateien lesen und ändern.
  </Accordion>

  <Accordion title="Generierten Code überprüfen">
    Überprüfen Sie immer den von der KI generierten Code, bevor Sie ihn in Ihr Projekt übernehmen.
  </Accordion>
</AccordionGroup>
