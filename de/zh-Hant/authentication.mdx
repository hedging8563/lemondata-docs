## API-Keys

Alle API-Anfragen erfordern eine Authentifizierung mittels API-Key. Bitte geben Sie Ihren Key im `Authorization`-Header an:

```bash
Authorization: Bearer sk-your-api-key
```

## Erhalt Ihres API-Keys

1. Melden Sie sich in Ihrem [LemonData Dashboard](https://lemondata.cc/dashboard) an
2. Navigieren Sie zum Bereich **API Keys**
3. Klicken Sie auf **Create New Key**
4. Vergeben Sie einen beschreibenden Namen für Ihren Key
5. Kopieren Sie den Key sofort – er wird nur einmal angezeigt

<Warning>
  **Best Practices für die Sicherheit:**
  - Geben Sie API-Keys niemals im clientseitigen Code preis
  - Übertragen Sie Keys nicht in Versionskontrollsysteme
  - Verwenden Sie Umgebungsvariablen zum Speichern von Keys
  - Rotieren Sie Ihre Keys regelmäßig
  - Löschen Sie nicht verwendete Keys
</Warning>

## Verwendung des API-Keys

<CodeGroup>

```bash cURL
curl https://api.lemondata.cc/v1/chat/completions \
  -H "Authorization: Bearer $LEMONDATA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model": "gpt-4o", "messages": [{"role": "user", "content": "Hello!"}]}'
```

```python Python
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ.get("LEMONDATA_API_KEY"),
    base_url="https://api.lemondata.cc/v1"
)
```

```javascript JavaScript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: process.env.LEMONDATA_API_KEY,
  baseURL: 'https://api.lemondata.cc/v1'
});
```

```go Go
package main

import "github.com/sashabaranov/go-openai"

func main() {
    config := openai.DefaultConfig(os.Getenv("LEMONDATA_API_KEY"))
    config.BaseURL = "https://api.lemondata.cc/v1"
    client := openai.NewClientWithConfig(config)
}
```

```php PHP
<?php
$apiKey = getenv('LEMONDATA_API_KEY');

$ch = curl_init('https://api.lemondata.cc/v1/chat/completions');
curl_setopt_array($ch, [
    CURLOPT_HTTPHEADER => [
        'Authorization: Bearer ' . $apiKey,
        'Content-Type: application/json'
    ]
]);
```

</CodeGroup>

## API-Key-Funktionen

### Nutzungslimits

Sie können für jeden API-Key Nutzungslimits festlegen, um Ihre Ausgaben zu kontrollieren:

| Einstellung | Beschreibung |
|---------|-------------|
| **Unbegrenzt** | Der Key verwendet Ihr Kontoguthaben ohne Einschränkungen |
| **Festes Limit** | Der Key funktioniert nicht mehr, sobald der festgelegte Betrag erreicht ist |

### Key-Präfix

Alle LemonData API-Keys beginnen mit dem Präfix `sk-`. Das Format des Keys ist:

```
sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## Anthropic API-Kompatibilität

Für den Endpunkt `/v1/messages` können Sie den Header `x-api-key` verwenden (kompatibel mit dem Anthropic SDK):

```bash
curl https://api.lemondata.cc/v1/messages \
  -H "x-api-key: sk-your-api-key" \
  -H "anthropic-version: 2023-06-01" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-5",
    "max_tokens": 1024,
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
```

<Note>
  Der Header `x-api-key` unterstützt nur die Endpunkte `/v1/messages` und `/v1/messages/count_tokens`. Andere Endpunkte erfordern den Header `Authorization: Bearer`.
</Note>

## Fehlerantworten

| Statuscode | Typ | Code | Beschreibung |
|-------------|------|------|-------------|
| 401 | `invalid_request_error` | `invalid_api_key` | Fehlender oder ungültiger API-Key |
| 401 | `invalid_request_error` | `expired_api_key` | API-Key wurde widerrufen |
| 402 | `insufficient_quota` | `insufficient_quota` | Unzureichendes Kontoguthaben |
| 402 | `insufficient_quota` | `quota_exceeded` | API-Key-Nutzungslimit erreicht |

Beispiel für eine Fehlerantwort:

```json
{
  "error": {
    "message": "Invalid API key provided",
    "type": "invalid_api_key",
    "code": "invalid_api_key"
  }
}
```
