---
title: "Dokumente reranken"
openapi: "POST /v1/rerank"
description: "Ordnet Dokumente basierend auf ihrer Relevanz für eine Suchanfrage neu"
---

Verwendet semantische Ähnlichkeitsmodelle, um Dokumente neu zu ordnen. Geeignet für die Optimierung von Suchergebnissen und RAG-Anwendungen.

## Request Body

<ParamField body="model" type="string" required>
  Die ID des zu verwendenden Reranking-Modells (z. B. `BAAI/bge-reranker-v2-m3`, `qwen3-rerank`).
</ParamField>

<ParamField body="query" type="string" required>
  Die Suchanfrage, die zur Einstufung der Dokumente verwendet wird.
</ParamField>

<ParamField body="documents" type="array" required>
  Eine Liste von Dokumenten (Strings), die neu geordnet werden sollen.
</ParamField>

<ParamField body="top_n" type="integer">
  Die Anzahl der zurückzugebenden Top-Ergebnisse. Standardmäßig werden alle Dokumente zurückgegeben.
</ParamField>

<ParamField body="return_documents" type="boolean" default="false">
  Gibt an, ob der ursprüngliche Dokumenttext in der Antwort enthalten sein soll.
</ParamField>

## Antwort

<ResponseField name="results" type="array">
  Eine sortierte Liste von Dokumenten mit Bewertungen.

  Jedes Ergebnis enthält:
  - `index` (integer): Index des ursprünglichen Dokuments
  - `relevance_score` (number): Relevanz-Score (0-1)
  - `document` (string): Ursprünglicher Text (falls `return_documents=true`)
</ResponseField>

<ResponseField name="model" type="string">
  Das für das Reranking verwendete Modell.
</ResponseField>

<ResponseField name="usage" type="object">
  Statistiken zur Token-Nutzung.
</ResponseField>
