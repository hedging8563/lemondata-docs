---
title: "Chat-Completion erstellen (Create Chat Completion)"
openapi: "POST /v1/chat/completions"
description: "Erstellt eine Vervollständigung für Chat-Nachrichten"
---

## Anfrage-Body (Request Body)

<ParamField body="model" type="string" required>
  Die ID des zu verwendenden Modells. Siehe [Models](https://lemondata.cc/zh-TW/models) für verfügbare Optionen.
</ParamField>

<ParamField body="messages" type="array" required>
  Eine Liste von Nachrichten, die die Konversation bilden.

  Jedes Nachrichtenobjekt enthält:
  - `role` (string): `system`, `user` oder `assistant`
  - `content` (string | array): Inhalt der Nachricht
</ParamField>

<ParamField body="temperature" type="number" default="1">
  Sampling-Temperatur zwischen 0 und 2. Höhere Werte machen die Ausgabe zufälliger.
</ParamField>

<ParamField body="max_tokens" type="integer">
  Die maximale Anzahl der zu generierenden Tokens.
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  Wenn true, werden Teil-Nachrichten-Inkremente als SSE-Events gesendet.
</ParamField>

<ParamField body="stream_options" type="object">
  Streaming-Optionen. Setzen Sie `include_usage: true`, um die Token-Nutzung in den Streaming-Chunks zu erhalten.
</ParamField>

<ParamField body="top_p" type="number" default="1">
  Parameter für Nucleus-Sampling. Wir empfehlen, entweder diesen Parameter oder die temperature anzupassen, aber nicht beide gleichzeitig.
</ParamField>

<ParamField body="frequency_penalty" type="number" default="0">
  Ein Wert zwischen -2,0 und 2,0. Positive Werte bestrafen wiederholte Tokens.
</ParamField>

<ParamField body="presence_penalty" type="number" default="0">
  Ein Wert zwischen -2,0 und 2,0. Positive Werte bestrafen Tokens, die bereits im Text vorkommen.
</ParamField>

<ParamField body="stop" type="string | array">
  Bis zu 4 Sequenzen, bei denen die API die Generierung von Tokens stoppt.
</ParamField>

<ParamField body="tools" type="array">
  Eine Liste von Tools, die das Modell aufrufen kann (function calling).
</ParamField>

<ParamField body="tool_choice" type="string | object">
  Steuert, wie das Modell Tools verwendet. Optionen: `auto`, `none`, `required` oder ein spezifisches Tool-Objekt.
</ParamField>

<ParamField body="parallel_tool_calls" type="boolean" default="true">
  Gibt an, ob parallele Funktionsaufrufe aktiviert sind. Bei false werden Funktionen nacheinander aufgerufen.
</ParamField>

<ParamField body="max_completion_tokens" type="integer">
  Maximale Anzahl der Tokens für die Vervollständigung. Eine Alternative zu `max_tokens`, empfohlen für neuere Modelle wie o1/o3.
</ParamField>

<ParamField body="reasoning_effort" type="string">
  Die Reasoning-Intensität für o1/o3 Modelle. Optionen: `low`, `medium`, `high`.
</ParamField>

<ParamField body="seed" type="integer">
  Ein Zufallssame für deterministisches Sampling.
</ParamField>

<ParamField body="n" type="integer" default="1">
  Die Anzahl der zu generierenden Vervollständigungen (1-128).
</ParamField>

<ParamField body="logprobs" type="boolean">
  Gibt an, ob Log-Wahrscheinlichkeiten (log probabilities) zurückgegeben werden sollen.
</ParamField>

<ParamField body="top_logprobs" type="integer">
  Die Anzahl der am häufigsten vorkommenden Log-Wahrscheinlichkeiten, die zurückgegeben werden sollen (0-20). Erfordert `logprobs: true`.
</ParamField>

<ParamField body="top_k" type="integer">
  Top-K Sampling-Parameter (für Anthropic/Gemini Modelle).
</ParamField>

<ParamField body="response_format" type="object">
  Spezifikation des Antwortformats. Verwenden Sie `{"type": "json_object"}`, um den JSON-Modus zu aktivieren, oder `{"type": "json_schema", "json_schema": {...}}` für eine strukturierte Ausgabe.
</ParamField>

<ParamField body="logit_bias" type="object">
  Modifiziert die Wahrscheinlichkeit des Auftretens bestimmter Tokens. Ordnet Token-IDs (als String) einem Bias-Wert zwischen -100 und 100 zu.
</ParamField>

<ParamField body="user" type="string">
  Eine eindeutige Kennung für Ihren Endbenutzer, die zur Missbrauchsüberwachung verwendet wird.
</ParamField>

<ParamField body="cache_control" type="object">
  LemonData Cache-Steuerungsoptionen.

  - `type` (string): Cache-Strategie - `default`, `no_cache`, `no_store`, `response_only`, `semantic_only`
  - `max_age` (integer): Cache-Lebensdauer (TTL) in Sekunden (maximal 86400)
</ParamField>

## Antwort (Response)

<ResponseField name="id" type="string">
  Eindeutige Kennung für die Vervollständigung.
</ResponseField>

<ResponseField name="object" type="string">
  Immer `chat.completion`.
</ResponseField>

<ResponseField name="created" type="integer">
  Der Unix-Zeitstempel der Erstellung der Vervollständigung.
</ResponseField>

<ResponseField name="model" type="string">
  Das für die Vervollständigung verwendete Modell.
</ResponseField>

<ResponseField name="choices" type="array">
  Eine Liste von Vervollständigungsoptionen.

  Jede Option enthält:
  - `index` (integer): Index der Option
  - `message` (object): Die generierte Nachricht
  - `finish_reason` (string): Der Grund, warum das Modell gestoppt hat (`stop`, `length`, `tool_calls`)
</ResponseField>

<ResponseField name="usage" type="object">
  Token-Nutzungsstatistiken.

  - `prompt_tokens` (integer): Anzahl der Tokens im Prompt
  - `completion_tokens` (integer): Anzahl der Tokens in der Vervollständigung
  - `total_tokens` (integer): Gesamtanzahl der verwendeten Tokens
</ResponseField>

from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.lemondata.cc/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Hello!"}
    ],
    temperature=0.7,
    max_tokens=1000
)

print(response.choices[0].message.content)

```go Go
package main

import (
    "context"
    "fmt"
    "github.com/sashabaranov/go-openai"
)

func main() {
    config := openai.DefaultConfig("sk-your-api-key")
    config.BaseURL = "https://api.lemondata.cc/v1"
    client := openai.NewClientWithConfig(config)

    resp, err := client.CreateChatCompletion(
        context.Background(),
        openai.ChatCompletionRequest{
            Model: "gpt-4o",
            Messages: []openai.ChatCompletionMessage{
                {Role: openai.ChatMessageRoleSystem, Content: "You are a helpful assistant."},
                {Role: openai.ChatMessageRoleUser, Content: "Hello!"},
            },
            Temperature: 0.7,
            MaxTokens:   1000,
        },
    )
    if err != nil {
        panic(err)
    }
    fmt.Println(resp.Choices[0].Message.Content)
}
```
