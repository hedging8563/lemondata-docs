## 개요

Dify는 오픈 소스 LLM 애플리케이션 개발 플랫폼입니다. 시각적 프롬프트 오케스트레이션, RAG 파이프라인, 에이전트 프레임워크 및 LLMOps 기능을 제공합니다. LemonData는 Dify에서 커스텀 모델 제공업체로 설정할 수 있습니다.

## 주요 이점

- 하나의 인터페이스를 통해 300개 이상의 AI 모델에 액세스
- 애플리케이션 로직을 변경하지 않고 모델 간 전환
- 각 작업에 가장 적합한 모델을 선택하여 비용 최적화
- 통합된 빌링 및 사용량 추적

## 사전 요구 사항

- API 액세스 권한이 있는 LemonData 계정
- Dify 설치 (클라우드 또는 셀프 호스팅)

## 설정 단계

### 1단계: API Key 받기

1. [LemonData Dashboard](https://lemondata.cc/dashboard)에 로그인합니다
2. [API Keys](https://lemondata.cc/dashboard/api)로 이동합니다
3. API Key를 생성하고 복사합니다 (형식: `sk-...`)

### 2단계: 커스텀 모델 제공업체 추가

<Steps>
  <Step title="설정 열기">
    Dify에서 **Settings** → **Model Provider**로 이동합니다
  </Step>
  <Step title="OpenAI 호환 제공업체 추가">
    **Add Model Provider**를 클릭하고 **OpenAI-API-compatible**을 선택합니다
  </Step>
  <Step title="제공업체 설정">
    다음 설정을 입력합니다:

    | 필드 | 값 |
    |-------|-------|
    | 제공업체 이름 | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="모델 추가">
    사용하려는 모델을 추가합니다:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### 3단계: 연결 테스트

1. 모델을 선택합니다 (예: `gpt-4o-mini`)
2. 테스트 메시지를 보냅니다
3. 응답을 받는지 확인합니다

## 애플리케이션에서 사용하기

### 챗봇

1. 새로운 챗봇 애플리케이션을 생성합니다
2. 모델 제공업체로 LemonData를 선택합니다
3. 선호하는 모델을 선택합니다
4. 시스템 프롬프트 및 파라미터를 설정합니다

### 에이전트

1. 에이전트 애플리케이션을 생성합니다
2. 성능이 뛰어난 모델(GPT-4o, Claude)을 선택합니다
3. 도구 및 지식 베이스를 추가합니다
4. 에이전트 동작을 설정합니다

### 워크플로우

1. 워크플로우를 생성합니다
2. LLM 노드를 추가합니다
3. 각 노드에 대해 LemonData 모델을 선택합니다
4. 노드를 연결하고 데이터 흐름을 설정합니다

## 사용 가능한 모델

| 카테고리 | 모델 |
|----------|--------|
| 채팅 | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| 임베딩 | text-embedding-3-small, text-embedding-3-large |
| 비전 | GPT-4o (이미지 포함), Claude Sonnet (이미지 포함) |

## RAG 설정

RAG 애플리케이션의 경우 임베딩을 설정하세요:

1. **Settings** → **Model Provider**로 이동합니다
2. 임베딩 모델 추가: `text-embedding-3-small`
3. 지식 베이스 설정에서 기본 임베딩 모델로 설정합니다

## 문제 해결

<AccordionGroup>
  <Accordion title="연결 오류">
    - API Base URL이 정확히 `https://api.lemondata.cc/v1`인지 확인하세요
    - 끝에 슬래시(/)가 있는지 확인하세요
    - 네트워크 연결 상태를 확인하세요
  </Accordion>

  <Accordion title="401 Unauthorized (권한 없음)">
    - API Key를 다시 확인하세요
    - LemonData 대시보드에서 키가 활성 상태인지 확인하세요
  </Accordion>

  <Accordion title="모델을 찾을 수 없음">
    - 모델 이름이 정확히 일치하는지 확인하세요
    - [lemondata.cc/en/models](https://lemondata.cc/ko/models)에서 모델 가용성을 확인하세요
  </Accordion>
</AccordionGroup>
