WeCom(企业微信 / WeChat Work)을 Claw 인스턴스와 통합하여 기업용 워크스페이스에 AI를 도입하세요.

## 사전 요구 사항

- [WeCom](https://work.weixin.qq.com/) 관리자 계정
- Claw 인스턴스 ([여기에서 생성](https://lemondata.cc/dashboard/claw))

## 설정 가이드

<Steps>
  <Step title="WeCom 앱 생성">
    1. [WeCom 관리자 콘솔](https://work.weixin.qq.com/wework_admin/frame)에 로그인합니다
    2. **앱 관리** → **앱 생성**으로 이동합니다
    3. 앱 이름을 입력하고, 로고를 업로드한 뒤, 가시 범위를 설정합니다
    4. **생성**을 클릭합니다
  </Step>
  <Step title="자격 증명 가져오기">
    1. 하단의 **내 기업** → **기업 정보**에서 **Corp ID**를 확인합니다
    2. 앱 설정에서 **Secret** (Corp Secret)을 복사합니다
  </Step>
  <Step title="Callback URL 구성">
    1. 앱 설정에서 **메시지 수신** → **API 수신 설정**으로 이동합니다
    2. WeCom 섹션을 확장하면 LemonData 대시보드에 callback URL이 표시됩니다:
    ```
    https://claw-{your-subdomain}.lemondata.cc/webhook/wecom
    ```
    3. **Token**(임의의 문자열)과 **EncodingAESKey**(43자리의 임의 문자열)를 설정합니다
    4. **저장**을 클릭합니다 — WeCom이 callback URL을 확인합니다

    <Tip>
    callback URL은 LemonData 대시보드에 복사 버튼과 함께 표시됩니다. URL을 확인하기 전에 Claw 인스턴스가 실행 중인지 확인하세요.
    </Tip>
  </Step>
  <Step title="LemonData 대시보드에서 구성">
    1. [Claw 대시보드](https://lemondata.cc/dashboard/claw)로 이동합니다
    2. 인스턴스의 **설정 아이콘**을 클릭합니다
    3. **IM 통합** 탭을 엽니다
    4. **WeCom** 섹션을 확장합니다
    5. 상단에 표시된 **Callback URL**을 복사합니다
    6. 다음 항목을 입력합니다: **Corp ID**, **Corp Secret**, **Token**, **EncodingAESKey**
    7. **저장**을 클릭합니다
  </Step>
  <Step title="채팅 시작하기">
    1. WeCom의 **워크벤치**에서 생성한 앱을 찾습니다
    2. 앱에 메시지를 보냅니다
    3. AI가 직접 응답합니다 — WeCom의 경우 별도의 페어링이 필요하지 않습니다
  </Step>
</Steps>

<Warning>
**Token**과 **EncodingAESKey**는 WeCom 관리자 콘솔과 Claw 설정 간에 정확히 일치해야 합니다. 일치하지 않으면 메시지 확인에 실패합니다.
</Warning>

## 문제 해결

<AccordionGroup>
  <Accordion title="Callback URL 확인 실패">
    - Claw 인스턴스가 실행 중인지 확인하세요(녹색 상태)
    - URL 형식을 확인하세요: `https://claw-{subdomain}.lemondata.cc/webhook/wecom`
    - WeCom에서 확인하기 **전에** Claw 설정에 Token과 EncodingAESKey가 저장되어 있는지 확인하세요
  </Accordion>
  <Accordion title="봇이 응답하지 않음">
    - Corp ID 및 Corp Secret이 올바른지 확인하세요
    - 사용자 계정에서 앱이 보이는지 확인하세요(가시 범위 설정)
    - WeCom과 Claw 간에 Token 및 EncodingAESKey가 일치하는지 확인하세요
  </Accordion>
</AccordionGroup>
