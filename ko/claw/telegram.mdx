Telegram은 연결하기 가장 **간단한** 플랫폼입니다. BotFather로부터 봇 토큰만 받으면 2분 안에 AI와 대화를 시작할 수 있습니다.

## 사전 요구 사항

- [Telegram](https://telegram.org/) 계정
- Claw 인스턴스 ([여기에서 생성](https://lemondata.cc/dashboard/claw))

## 설정 가이드

<Steps>
  <Step title="BotFather로 봇 생성하기">
    1. Telegram을 열고 **@BotFather**를 검색하세요
    2. `/newbot`을 전송하세요
    3. 봇의 **표시 이름**을 선택하세요 (예: "My AI Assistant")
    4. `bot`으로 끝나는 **사용자 이름**을 선택하세요 (예: `my_ai_assistant_bot`)
    5. BotFather가 **Bot Token**을 회신합니다. 다음과 같은 형식입니다:
    ```
    123456789:ABCdefGHIjklMNOpqrsTUVwxyz
    ```
    <Warning>
    봇 토큰을 비밀로 유지하세요! 이 토큰을 가진 사람은 누구나 봇을 제어할 수 있습니다.
    </Warning>
  </Step>
  <Step title="LemonData 대시보드에서 설정하기">
    1. [Claw 대시보드](https://lemondata.cc/dashboard/claw)로 이동하세요
    2. 인스턴스의 **설정 아이콘**을 클릭하세요
    3. **IM Integrations** 탭을 엽니다
    4. **Telegram** 섹션을 확장합니다
    5. **Bot Token**을 붙여넣으세요
    6. **Save**를 클릭하세요
  </Step>
  <Step title="대화 시작하기">
    1. Telegram을 열고 사용자 이름으로 봇을 찾으세요
    2. 아무 메시지나 전송하세요 (예: "Hello!")
    3. 첫 번째 사용자가 자동으로 소유자로 **페어링**됩니다 — 코드가 필요하지 않습니다!
    4. AI와 대화를 시작하세요!
  </Step>
</Steps>

## 선택 사항: 봇 프로필 사용자 정의

봇을 생성한 후 BotFather를 사용하여 사용자 정의할 수 있습니다:

| 명령어 | 용도 |
|---------|---------|
| `/setdescription` | 봇의 바이오/설명 설정 |
| `/setabouttext` | "정보" 섹션 설정 |
| `/setuserpic` | 프로필 사진 업로드 |
| `/setcommands` | 명령어 제안 추가 |

## 그룹에서 사용하기

<Warning>
Telegram 봇은 **Privacy Mode가 기본적으로 활성화**되어 있습니다. 그룹 채팅에서 봇은 다음 메시지만 볼 수 있습니다:
- `/`로 시작하는 메시지 (명령어)
- 봇을 직접 @mention 하는 메시지
- 봇의 메시지에 대한 답장

봇이 **모든** 그룹 메시지를 볼 수 있게 하려면 BotFather를 통해 Privacy Mode를 비활성화하세요: `/setprivacy` 전송 → 봇 선택 → **Disable** 선택. 참고: 이 변경 사항을 적용하려면 기존 그룹에서 봇을 **제거했다가 다시 추가**해야 합니다.
</Warning>

### 그룹 설정

대시보드의 **IM Integrations** 탭에서 그룹 동작을 설정하세요:

| 설정 | 옵션 | 설명 |
|---------|---------|-------------|
| **Group Policy** | Open (기본값) / Allowlist / Disabled | 그룹에서 누가 봇을 트리거할 수 있는지 제어합니다 |
| **Require @mention** | On / Off | 봇이 @mention 되었을 때만 응답합니다 (권장) |
| **Group IDs** | 채팅 ID 목록 | 특정 그룹으로 제한 (비어 있음 = 모든 그룹) |
| **Group Allowed Users** | 사용자 ID 목록 | 이 사용자들만 봇을 트리거할 수 있습니다 (allowlist 모드) |

<Tip>
**Group Policy**는 그룹에서 *누가* 봇에게 메시지를 보낼 수 있는지 제어하고, **Group IDs**는 봇이 *어떤* 그룹에서 응답할지 제어합니다. 세밀한 제어를 위해 두 설정을 함께 사용하세요.
</Tip>

## DM 정책

누가 봇에게 다이렉트 메시지를 보낼 수 있는지 제어합니다:

| 정책 | 동작 |
|--------|----------|
| **Pairing** (기본값) | 새 사용자는 페어링 코드를 받습니다 — 승인을 위해 WebUI에 입력하세요 |
| **Allowlist** | 지정한 사용자 ID만 봇에게 메시지를 보낼 수 있습니다 |
| **Open** | 누구나 봇에게 메시지를 보낼 수 있습니다 (주의해서 사용하세요 — AI 크레딧이 소모됩니다) |

## 문제 해결

<AccordionGroup>
  <Accordion title="봇이 응답하지 않음">
    - 봇 토큰이 정확한지 확인하세요 (공백 없음)
    - 인스턴스가 실행 중인지 확인하세요 (대시보드의 녹색 상태)
    - 대시보드에서 인스턴스를 재시작해 보세요
  </Accordion>
  <Accordion title="'페어링 코드'가 나타나지 않음">
    - 그룹이 아닌 봇에게 **다이렉트 메시지**를 보내고 있는지 확인하세요
    - 페어링 시스템은 새 사용자의 첫 번째 메시지에서 활성화됩니다
  </Accordion>
  <Accordion title="봇의 응답이 느림">
    - 응답 시간은 선택한 AI 모델에 따라 다릅니다
    - 더 큰 모델(Claude Opus, GPT-5)은 몇 초 정도 걸릴 수 있습니다
    - LemonData 잔액을 확인하세요 — 크레딧이 부족하면 오류가 발생합니다
  </Accordion>
</AccordionGroup>
