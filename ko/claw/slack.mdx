Socket Mode를 사용하여 Slack 워크스페이스에 AI 어시스턴트를 추가하세요 — 공개 URL이 필요하지 않습니다.

## 사전 요구 사항

- 관리자 권한이 있는 [Slack](https://slack.com/) 워크스페이스
- Claw 인스턴스 ([여기에서 생성](https://lemondata.cc/dashboard/claw))

## 설정 가이드

<Steps>
  <Step title="Slack 앱 생성">
    1. [Slack API](https://api.slack.com/apps)로 이동하여 **Create New App**을 클릭합니다
    2. **From scratch**를 선택합니다
    3. 앱 이름(예: "AI Assistant")을 입력하고 워크스페이스를 선택합니다
    4. **Create App**을 클릭합니다
  </Step>
  <Step title="Socket Mode 활성화">
    1. 왼쪽 사이드바에서 **Socket Mode**로 이동합니다
    2. **Enable Socket Mode**를 ON으로 토글합니다
    3. 토큰 이름(예: "claw-socket")을 지정하고 **Generate**를 클릭합니다
    4. **App-Level Token**(`xapp-`으로 시작)을 복사합니다 — 이 토큰이 필요합니다
  </Step>
  <Step title="봇 권한 구성">
    1. 사이드바에서 **OAuth & Permissions**로 이동합니다
    2. **Bot Token Scopes** 아래에 다음 스코프를 추가합니다:
       - `chat:write` — 메시지 전송
       - `im:history` — DM 기록 읽기
       - `im:read` — DM 보기
       - `im:write` — DM 시작
       - `app_mentions:read` — @멘션 감지
    3. 상단의 **Install to Workspace**를 클릭합니다
    4. 앱을 승인합니다
    5. **Bot User OAuth Token**(`xoxb-`으로 시작)을 복사합니다
  </Step>
  <Step title="이벤트 활성화">
    1. 사이드바에서 **Event Subscriptions**로 이동합니다
    2. **Enable Events**를 ON으로 토글합니다
    3. **Subscribe to bot events** 아래에 다음을 추가합니다:
       - `message.im` — DM 메시지
       - `app_mention` — 채널 내 @멘션
    4. **Save Changes**를 클릭합니다
  </Step>
  <Step title="LemonData 대시보드에서 구성">
    1. [Claw 대시보드](https://lemondata.cc/dashboard/claw)로 이동합니다
    2. 인스턴스의 **설정 아이콘**을 클릭합니다
    3. **IM Integrations** 탭을 엽니다
    4. **Slack** 섹션을 확장합니다
    5. **App Token**(`xapp-...`)과 **Bot Token**(`xoxb-...`)을 붙여넣습니다
    6. **Save**를 클릭합니다
  </Step>
  <Step title="채팅 시작하기">
    1. Slack에서 봇에게 DM을 보내거나 채널에서 @멘션하세요
    2. 페어링 프로세스를 완료합니다
    3. AI와 채팅을 시작하세요!
  </Step>
</Steps>

## 접근 제어

### 그룹 설정

대시보드의 **IM Integrations** 탭에서 채널 접근을 구성합니다:

| 설정 | 옵션 | 설명 |
|---------|---------|-------------|
| **Group Policy** | Open (기본값) / Allowlist / Disabled | 봇이 응답하는 채널을 제어합니다 |
| **Require @mention** | On / Off | @멘션된 경우에만 봇이 응답합니다 |
| **Group IDs** | 채널 ID 목록 | 특정 채널로 제한합니다 (비어 있으면 = 모든 채널) |
| **Group Allowed Users** | 사용자 ID 목록 | 이 사용자만 봇을 트리거할 수 있습니다 (allowlist 모드) |

<Tip>
기본적으로 Slack은 **open** 그룹 정책을 사용합니다 — 봇이 초대된 모든 채널에서 응답합니다. 특정 채널로 제한하려면 **allowlist**를 사용하세요.
</Tip>

## 문제 해결

<AccordionGroup>
  <Accordion title="봇이 DM에 응답하지 않음">
    - 두 토큰이 모두 올바른지 확인하세요 (App Token은 `xapp-`으로 시작, Bot Token은 `xoxb-`으로 시작)
    - `message.im` 이벤트가 구독되어 있는지 확인하세요
    - 앱이 워크스페이스에 설치되어 있는지 확인하세요
  </Accordion>
  <Accordion title="봇이 @멘션에 응답하지 않음">
    - `app_mentions:read` 스코프와 `app_mention` 이벤트가 구성되어 있는지 확인하세요
    - 봇을 먼저 채널에 초대해야 합니다 (`/invite @botname` 입력)
  </Accordion>
  <Accordion title="'not_allowed_token_type' 오류">
    - App Token 필드에 Bot Token이 아닌 **App-Level Token**(`xapp-`)을 사용하고 있는지 확인하세요
    - Bot Token(`xoxb-`)은 Bot Token 필드에 입력합니다
  </Accordion>
</AccordionGroup>
