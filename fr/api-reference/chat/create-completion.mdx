---
title: "Créer une complétion de chat"
openapi: "POST /v1/chat/completions"
description: "Crée une complétion pour le message de chat"
---

## Corps de la requête

<ParamField body="model" type="string" required>
  ID du modèle à utiliser. Voir [Modèles](https://lemondata.cc/fr/models) pour les options disponibles.
</ParamField>

<ParamField body="messages" type="array" required>
  Une liste de messages composant la conversation.

  Chaque objet message contient :
  - `role` (string) : `system`, `user`, ou `assistant`
  - `content` (string | array) : Le contenu du message
</ParamField>

<ParamField body="temperature" type="number" default="1">
  Température d'échantillonnage entre 0 et 2. Des valeurs plus élevées rendent la sortie plus aléatoire.
</ParamField>

<ParamField body="max_tokens" type="integer">
  Nombre maximum de tokens à générer.
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  Si vrai, des deltas de messages partiels seront envoyés sous forme d'événements SSE.
</ParamField>

<ParamField body="stream_options" type="object">
  Options pour le streaming. Définissez `include_usage: true` pour recevoir l'utilisation des tokens dans les morceaux du flux.
</ParamField>

<ParamField body="top_p" type="number" default="1">
  Paramètre d'échantillonnage nucleus. Nous recommandons de modifier ceci ou la température, mais pas les deux.
</ParamField>

<ParamField body="frequency_penalty" type="number" default="0">
  Nombre entre -2,0 et 2,0. Les valeurs positives pénalisent les tokens répétés.
</ParamField>

<ParamField body="presence_penalty" type="number" default="0">
  Nombre entre -2,0 et 2,0. Les valeurs positives pénalisent les tokens déjà présents dans le texte.
</ParamField>

<ParamField body="stop" type="string | array">
  Jusqu'à 4 séquences où l'API arrêtera de générer des tokens.
</ParamField>

<ParamField body="tools" type="array">
  Une liste d'outils que le modèle peut appeler (appel de fonction).
</ParamField>

<ParamField body="tool_choice" type="string | object">
  Contrôle la manière dont le modèle utilise les outils. Options : `auto`, `none`, `required`, ou un objet outil spécifique.
</ParamField>
