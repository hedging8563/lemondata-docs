<Note>
Ce skill apprend à votre agent de codage comment intégrer n'importe laquelle des **plus de 300 API d'IA** de LemonData en décrivant simplement ce dont vous avez besoin. Il fonctionne avec **Claude Code, Cursor, GitHub Copilot, Windsurf**, et tout outil prenant en charge les fichiers de skills/règles.

L'API est **[Agent-First](/guides/agent-first-api)** — même si l'agent se trompe de nom de modèle, la réponse d'erreur lui indique exactement comment corriger le problème.
</Note>

## Comment ça fonctionne

1. **Vous décrivez ce dont vous avez besoin** — "Je veux utiliser GPT-4 dans mon projet Python"
2. **L'agent appelle l'API** — il peut deviner un nom de modèle ou un endpoint
3. **Si quelque chose ne va pas** — l'API renvoie des indices structurés (`did_you_mean`, `suggestions`, `retry_after`)
4. **L'agent s'auto-corrige** — lit les indices et réessaie avec les bons paramètres
5. **Vous obtenez un code fonctionnel** — généralement dès la première ou deuxième tentative

## Obtenir votre clé API

<Steps>
  <Step title="Visitez LemonData">
    Allez sur [lemondata.cc](https://lemondata.cc)
  </Step>
  <Step title="Connectez-vous">
    Créez un compte ou connectez-vous
  </Step>
  <Step title="Obtenez la clé API">
    Accédez au [Dashboard → API Keys](https://lemondata.cc/dashboard/api) et créez une nouvelle clé

</Tip>

## Exemples

### Complétion de Chat

**Vous dites :**
```
I want to use GPT-4 in my Python project
```

**L'agent génère :**
```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.lemondata.cc/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

Si l'agent devine un mauvais nom de modèle (ex: `gpt5`), l'API renvoie :
```json
{
  "error": {
    "code": "model_not_found",
    "did_you_mean": "gpt-4o",
    "suggestions": [{"id": "gpt-4o"}, {"id": "gpt-4o-mini"}],
    "hint": "Did you mean 'gpt-4o'? Use GET /v1/models to list all available models."
  }
}
```

L'agent lit `did_you_mean`, corrige le nom du modèle et réessaie — le tout automatiquement.

### Génération d'images

**Vous dites :**
```
Generate images with Flux in Node.js
```

L'agent appelle l'API d'image, gère les erreurs et génère un code complet avec la logique de téléchargement et de sauvegarde.

### Reconnaissance vocale

**Vous dites :**
```
Integrate speech-to-text in my app
```

L'agent génère le code avec le bon endpoint `/v1/audio/transcriptions` et la gestion des fichiers audio.

## Récupération d'erreur Agent-First

Chaque réponse d'erreur inclut des indices structurés que n'importe quel agent de codage peut analyser. Consultez la [référence complète de l'API Agent-First](/guides/agent-first-api) pour plus de détails.

| Erreur | Ce que l'API renvoie | Ce que fait l'agent |
|-------|---------------------|---------------------|
| Nom de modèle erroné | `did_you_mean` + `suggestions` | S'auto-corrige et réessaie |
| Solde insuffisant | `balance_usd` + `suggestions` moins chères | Bascule vers un modèle abordable |
| Modèle indisponible | `alternatives` + `retry_after` | Bascule vers un modèle disponible |
| Limite de débit atteinte | `retry_after` (secondes exactes) | Attend et réessaie |
| Contexte trop long | `suggestions` avec des modèles plus grands | Bascule vers un modèle à plus large contexte |

### Optimisation native des endpoints

Lorsque vous appelez `/v1/chat/completions` avec un modèle Claude ou Gemini, l'API renvoie des en-têtes d'optimisation :

```
X-LemonData-Hint: This model supports native Anthropic format. Use POST /v1/messages for better performance.
X-LemonData-Native-Endpoint: /v1/messages
```

L'agent bascule automatiquement vers l'endpoint natif pour de meilleures performances (pas de conversion de format, pensée étendue, mise en cache des prompts).

### Découverte de modèles

Pas besoin de chercher dans la documentation :

```bash
# Aperçu de l'API lisible par machine
curl https://lemondata.cc/llms.txt

# Lister les modèles par catégorie
curl "https://api.lemondata.cc/v1/models?category=chat" -H "Authorization: Bearer sk-KEY"
curl "https://api.lemondata.cc/v1/models?category=image" -H "Authorization: Bearer sk-KEY"

# Filtrer par capacité
curl "https://api.lemondata.cc/v1/models?tag=coding" -H "Authorization: Bearer sk-KEY"
```

## FAQ

<AccordionGroup>
  <Accordion title="Le skill ne se déclenche pas automatiquement ?">
    Essayez de mentionner "LemonData" ou "API LemonData" dans votre requête :
    ```
    Use LemonData to integrate GPT-4 in my project
    ```
  </Accordion>

  <Accordion title="Comment l'API aide-t-elle l'agent à trouver le bon modèle ?">
    L'API [Agent-First](/guides/agent-first-api) de LemonData renvoie des indices d'erreur structurés. Si l'agent devine un mauvais nom de modèle, l'erreur inclut `did_you_mean` avec le nom correct et `suggestions` avec des alternatives. L'agent les lit et s'auto-corrige lors de la tentative suivante.
  </Accordion>

  <Accordion title="Quels agents de codage sont pris en charge ?">
    Tout outil prenant en charge les fichiers de skills/règles : Claude Code, Cursor, GitHub Copilot, Windsurf, et plus encore. L'installateur `npx add-skill` détecte et installe automatiquement sur tous les outils pris en charge.
  </Accordion>

  <Accordion title="Comment mettre à jour le skill ?">
    Relancez l'installateur :
    ```bash
    npx add-skill hedging8563/lemondata-api-skill -y
    ```
  </Accordion>
</AccordionGroup>
