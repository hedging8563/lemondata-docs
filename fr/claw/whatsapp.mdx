---
title: "WhatsApp"
description: "Connectez votre instance Claw à WhatsApp via l'appairage par code QR"
---

Connectez WhatsApp à votre instance Claw en scannant un code QR — tout comme WhatsApp Web.

## Prérequis

- Un téléphone avec [WhatsApp](https://www.whatsapp.com/) installé et un compte actif
- Une instance Claw ([créez-en une ici](https://lemondata.cc/dashboard/claw))

<Warning>
WhatsApp utilise votre **numéro de téléphone réel**. Les messages envoyés via le bot apparaîtront comme provenant de votre compte WhatsApp. Utilisez un numéro dédié si vous souhaitez une séparation.
</Warning>

<Warning>
**Les numéros VoIP ne sont pas pris en charge.** WhatsApp nécessite un vrai numéro de téléphone basé sur une carte SIM. Les numéros virtuels (Google Voice, Skype, etc.) ne fonctionnent pas pour l'appairage par QR code.
</Warning>

## Guide de configuration

<Steps>
  <Step title="Enregistrer la configuration WhatsApp">
    1. Accédez à votre [Claw Dashboard](https://lemondata.cc/dashboard/claw)
    2. Cliquez sur l'**icône des paramètres** de votre instance
    3. Ouvrez l'onglet **IM Integrations**
    4. Développez la section **WhatsApp**

    Cliquez d'abord sur **Enregistrer** — vous devez enregistrer la configuration avant de scanner le QR code
  </Step>
  <Step title="Scanner le code QR">
    1. Cliquez sur le bouton **Scan QR Code**
    2. Un code QR apparaîtra (valide pendant environ 60 secondes)
    3. Sur votre téléphone, ouvrez WhatsApp → **Réglages** → **Appareils connectés** → **Connecter un appareil**
    4. Scannez le code QR affiché à l'écran
    5. Attendez que la connexion s'établisse
  </Step>
  <Step title="Configurer la politique d'accès">
    WhatsApp utilise la politique de **allowlist** — seuls les numéros de téléphone que vous spécifiez peuvent interagir avec le bot IA.

    Ajoutez les numéros de téléphone autorisés dans le tableau de bord LemonData (format international, par ex. `+1234567890`).

    <Tip>
    Contrairement aux autres plateformes, WhatsApp ne prend pas en charge le mécanisme d'appairage. Vous devez explicitement ajouter les contacts autorisés.
    </Tip>
  </Step>
  <Step title="Commencer à discuter">
    Envoyez un message au numéro WhatsApp connecté à partir d'un contact autorisé — l'IA répondra !
  </Step>
</Steps>

## Paramètres de groupe

Les groupes WhatsApp sont pris en charge. Configurez le comportement des groupes dans l'onglet **IM Integrations** :

| Paramètre | Options | Description |
|---------|---------|-------------|
| **Group Policy** | Open / Allowlist / Disabled | Contrôle qui peut déclencher le bot dans les groupes |
| **Require @mention** | On / Off | Le bot ne répond que lorsqu'il est @mentionné |
| **Group IDs** | Liste de JIDs de groupe | Restreindre à des groupes spécifiques (vide = tous les groupes) |
| **Group Allowed Users** | Liste de numéros de téléphone | Seuls ces utilisateurs peuvent déclencher le bot (mode allowlist) |

<Tip>
Utilisez le format E.164 pour les numéros de téléphone dans la liste d'autorisation de groupe (par ex. `+1234567890`).
</Tip>

## Dépannage

<AccordionGroup>
  <Accordion title="Code QR expiré">
    Cliquez à nouveau sur **Scan QR Code** pour générer un nouveau code. Les codes QR expirent après environ 60 secondes.
  </Accordion>
  <Accordion title="WhatsApp déconnecté">
    WhatsApp peut déconnecter les appareils liés après une période d'inactivité. Scannez à nouveau le code QR pour vous reconnecter.
  </Accordion>
  <Accordion title="Les messages ne reçoivent pas de réponses de l'IA">
    - Vérifiez que le numéro de l'expéditeur figure dans votre allowlist (si vous utilisez la politique de allowlist)
    - Vérifiez que votre instance est en cours d'exécution
    - Vérifiez votre solde LemonData
  </Accordion>
</AccordionGroup>
