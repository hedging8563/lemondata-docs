Telegram est la plateforme la **plus simple** à connecter. Vous avez seulement besoin d'un bot token de BotFather, et vous discuterez avec l'IA en moins de 2 minutes.

## Prérequis

- Un compte [Telegram](https://telegram.org/)
- Une instance Claw ([créez-en une ici](https://lemondata.cc/dashboard/claw))

## Guide de configuration

<Steps>
  <Step title="Créer un bot avec BotFather">
    1. Ouvrez Telegram et recherchez **@BotFather**
    2. Envoyez `/newbot`
    3. Choisissez un **nom d'affichage** pour votre bot (ex: "Mon Assistant IA")
    4. Choisissez un **nom d'utilisateur** se terminant par `bot` (ex: `my_ai_assistant_bot`)
    5. BotFather répondra avec votre **Bot Token** — il ressemble à ceci :
    ```
    123456789:ABCdefGHIjklMNOpqrsTUVwxyz
    ```
    <Warning>
    Gardez votre bot token secret ! Toute personne possédant ce token peut contrôler votre bot.
    </Warning>
  </Step>
  <Step title="Configurer dans le tableau de bord LemonData">
    1. Accédez à votre [tableau de bord Claw](https://lemondata.cc/dashboard/claw)
    2. Cliquez sur l'**icône des paramètres** de votre instance
    3. Ouvrez l'onglet **IM Integrations**
    4. Développez la section **Telegram**
    5. Collez votre **Bot Token**
    6. Cliquez sur **Save**
  </Step>
  <Step title="Commencer à discuter">
    1. Ouvrez Telegram et trouvez votre bot par son nom d'utilisateur
    2. Envoyez n'importe quel message (ex: "Bonjour !")
    3. Le premier utilisateur est **automatiquement associé** en tant que propriétaire — aucun code n'est nécessaire !
    4. Commencez à discuter avec l'IA !
  </Step>
</Steps>

## Optionnel : Personnaliser le profil du bot

Après avoir créé votre bot, vous pouvez le personnaliser avec BotFather :

| Commande | Objectif |
|---------|---------|
| `/setdescription` | Définir la bio/description du bot |
| `/setabouttext` | Définir la section "À propos" |
| `/setuserpic` | Télécharger une photo de profil |
| `/setcommands` | Ajouter des suggestions de commandes |

## Utilisation dans les groupes

<Warning>
**Le mode Confidentialité est activé par défaut** pour les bots Telegram. Dans les discussions de groupe, le bot ne verra que les messages qui :
- Commencent par `/` (commandes)
- Mentionnent directement le bot avec @
- Sont des réponses aux messages du bot

Pour permettre au bot de voir **tous** les messages du groupe, désactivez le mode Confidentialité via BotFather : envoyez `/setprivacy` → sélectionnez votre bot → choisissez **Disable**. Note : vous devez **supprimer et réajouter** le bot aux groupes existants pour que ce changement prenne effet.
</Warning>

### Paramètres de groupe

Configurez le comportement du groupe dans l'onglet **IM Integrations** de votre tableau de bord :

| Paramètre | Options | Description |
|---------|---------|-------------|
| **Group Policy** | Open (par défaut) / Allowlist / Disabled | Contrôle qui peut déclencher le bot dans les groupes |
| **Require @mention** | On / Off | Le bot ne répond que lorsqu'il est mentionné avec @ (recommandé) |
| **Group IDs** | Liste d'IDs de chat | Restreindre à des groupes spécifiques (vide = tous les groupes) |
| **Group Allowed Users** | Liste d'IDs d'utilisateurs | Seuls ces utilisateurs peuvent déclencher le bot (mode allowlist) |

<Tip>
**Group Policy** contrôle *qui* peut envoyer des messages au bot dans les groupes, tandis que **Group IDs** contrôle dans *quels* groupes le bot répond. Utilisez les deux ensemble pour un contrôle précis.
</Tip>

## Politique de DM

Contrôlez qui peut envoyer des messages directs à votre bot :

| Politique | Comportement |
|--------|----------|
| **Pairing** (par défaut) | Les nouveaux utilisateurs reçoivent un code d'association — saisissez-le dans la WebUI pour autoriser |
| **Allowlist** | Seuls les IDs d'utilisateurs que vous spécifiez peuvent envoyer des messages au bot |
| **Open** | N'importe qui peut envoyer des messages au bot (à utiliser avec précaution — consomme vos crédits IA) |

## Dépannage

<AccordionGroup>
  <Accordion title="Le bot ne répond pas">
    - Vérifiez que le bot token est correct (pas d'espaces supplémentaires)
    - Vérifiez que votre instance est en cours d'exécution (statut vert dans le tableau de bord)
    - Essayez de redémarrer l'instance depuis le tableau de bord
  </Accordion>
  <Accordion title="Le 'code d'association' n'apparaît pas">
    - Assurez-vous d'envoyer un **message direct** au bot, pas dans un groupe
    - Le système d'association s'active au premier message d'un nouvel utilisateur
  </Accordion>
  <Accordion title="Le bot répond lentement">
    - Le temps de réponse dépend du modèle d'IA sélectionné
    - Les modèles plus volumineux (Claude Opus, GPT-5) peuvent prendre quelques secondes
    - Vérifiez votre solde LemonData — des crédits insuffisants provoqueront des erreurs
  </Accordion>
</AccordionGroup>
