## Aperçu

Dify est une plateforme de développement d'applications LLM open-source. Elle propose une orchestration visuelle de Prompt, des pipelines RAG, un framework d'Agent ainsi que des capacités LLMOps. LemonData peut être configuré comme un fournisseur de modèles personnalisé dans Dify.

## Avantages

- Accès à plus de 300 modèles AI via une interface unique
- Changement de modèle sans modifier la logique de l'application
- Sélection du meilleur modèle pour chaque tâche afin d'optimiser les coûts
- Facturation et suivi de l'utilisation unifiés

## Prérequis

- Un compte LemonData avec accès API
- Une installation de Dify (version Cloud ou auto-hébergée)

## Étapes de configuration

### Étape 1 : Obtenir votre API Key

1. Connectez-vous au [LemonData Dashboard](https://lemondata.cc/dashboard)
2. Naviguez vers [API Keys](https://lemondata.cc/dashboard/api)
3. Créez et copiez votre API Key (format : `sk-...`)

### Étape 2 : Ajouter un fournisseur de modèles personnalisé

<Steps>
  <Step title="Ouvrir les paramètres">
    Dans Dify, allez dans **Settings** → **Model Provider**
  </Step>
  <Step title="Ajouter un fournisseur compatible OpenAI">
    Cliquez sur **Add Model Provider** et choisissez **OpenAI-API-compatible**
  </Step>
  <Step title="Configurer le fournisseur">
    Saisissez les paramètres suivants :

    | Champ | Valeur |
    |-------|-------|
    | Nom du fournisseur | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="Ajouter des modèles">
    Ajoutez les modèles que vous souhaitez utiliser :
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### Étape 3 : Tester la connexion

1. Sélectionnez un modèle (par exemple `gpt-4o-mini`)
2. Envoyez un message de test
3. Confirmez que vous recevez une réponse

## Utilisation dans les applications

### Chatbot

1. Créez une nouvelle application Chatbot
2. Sélectionnez LemonData comme fournisseur de modèles
3. Choisissez votre modèle préféré
4. Configurez le Prompt système et les paramètres

### Agent

1. Créez une application Agent
2. Choisissez un modèle performant (GPT-4o, Claude)
3. Ajoutez des outils et une base de connaissances
4. Configurez le comportement de l'Agent

### Workflow

1. Créez un workflow
2. Ajoutez un nœud LLM
3. Sélectionnez un modèle LemonData pour chaque nœud
4. Connectez les nœuds et configurez le flux de données

## Modèles disponibles

| Catégorie | Modèle |
|----------|--------|
| Conversation (Chat) | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| Embeddings | text-embedding-3-small, text-embedding-3-large |
| Vision | GPT-4o (supporte les images), Claude Sonnet (supporte les images) |

## Configuration RAG

Pour les applications RAG, veuillez configurer le modèle d'embedding :

1. Allez dans **Settings** → **Model Provider**
2. Ajoutez le modèle d'embedding : `text-embedding-3-small`
3. Définissez-le comme modèle d'embedding par défaut dans les paramètres de la base de connaissances

## Bonnes pratiques

<AccordionGroup>
  <Accordion title="Commencer par des modèles économiques">
    Utilisez GPT-4o-mini pendant les phases de test et de développement, puis passez à des modèles plus puissants en production.
  </Accordion>

  <Accordion title="Utiliser le routage de modèles">
    Configurez différents modèles pour différentes tâches — utilisez des modèles rapides pour les requêtes simples et des modèles puissants pour le raisonnement complexe.
  </Accordion>

  <Accordion title="Surveiller l'utilisation">
    Combinez les fonctionnalités LLMOps de Dify avec la console LemonData pour suivre les coûts et les performances.
  </Accordion>

## Dépannage

<AccordionGroup>
  <Accordion title="Erreur de connexion">
    - Vérifiez que l'API Base URL est exactement `https://api.lemondata.cc/v1`
    - Vérifiez s'il y a un slash à la fin
    - Vérifiez votre connexion réseau
  </Accordion>

  <Accordion title="401 Non autorisé (Unauthorized)">
    - Vérifiez à nouveau votre API Key
    - Assurez-vous que la clé est active dans la console LemonData
  </Accordion>

  <Accordion title="Modèle non trouvé">
    - Confirmez que le nom du modèle correspond exactement
    - Vérifiez la disponibilité du modèle sur [lemondata.cc/zh-TW/models](https://lemondata.cc/zh-TW/models)
  </Accordion>
</AccordionGroup>
