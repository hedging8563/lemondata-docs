## Formato de respuesta de error

Todos los errores devuelven un formato JSON consistente:

```json
{
  "error": {
    "message": "Human-readable error description",
    "type": "error_type",
    "code": "error_code",
    "param": "parameter_name"  // 選填，用於驗證錯誤
  }
}
```

## Códigos de estado HTTP

| Código | Descripción |
|------|-------------|
| 400 | Bad Request - Parámetros inválidos |
| 401 | Unauthorized - API key inválida o ausente |
| 402 | Payment Required - Saldo insuficiente |
| 403 | Forbidden - Acceso denegado o el modelo no está permitido |
| 404 | Not Found - Modelo o recurso no encontrado |
| 413 | Payload Too Large - El tamaño de la entrada o del archivo excede el límite |
| 429 | Too Many Requests - Límite de velocidad excedido |
| 500 | Internal Server Error - Error interno del servidor |
| 502 | Bad Gateway - Error del proveedor upstream |
| 503 | Service Unavailable - Todos los canales fallaron |
| 504 | Gateway Timeout - Tiempo de espera de la solicitud agotado |

## Tipos de errores

### Errores de autenticación (401)

| Tipo | Código | Descripción |
|------|------|-------------|
| `invalid_api_key` | `invalid_api_key` | API key ausente o inválida |
| `expired_api_key` | `expired_api_key` | API key revocada |

```python
from openai import OpenAI, AuthenticationError

try:
    response = client.chat.completions.create(...)
except AuthenticationError as e:
    print(f"Authentication failed: {e.message}")
```

### Errores de pago (402)

| Tipo | Código | Descripción |
|------|------|-------------|
| `insufficient_quota` | `insufficient_quota` | Saldo de la cuenta demasiado bajo |
| `quota_exceeded` | `quota_exceeded` | Se ha alcanzado el límite de uso de la API key |

```python
from openai import OpenAI, APIStatusError

try:
    response = client.chat.completions.create(...)
except APIStatusError as e:
    if e.status_code == 402:
        print("Please top up your account balance")
```

### Errores de acceso (403)

| Tipo | Código | Descripción |
|------|------|-------------|
| `access_denied` | `access_denied` | Acceso al recurso denegado |
| `access_denied` | `model_not_allowed` | Esta API key no tiene permitido usar este modelo |

```json
{
  "error": {
    "message": "You don't have permission to access this model",
    "type": "access_denied",
    "code": "model_not_allowed"
  }
}
```

### Errores de validación (400)

| Tipo | Descripción |
|------|-------------|
| `invalid_request_error` | Parámetros de solicitud inválidos |
| `context_length_exceeded` | El contenido de entrada es demasiado largo para el modelo |
| `model_not_found` | El modelo solicitado no existe |

```json
{
  "error": {
    "message": "model: 'invalid-model' is not a valid model",
    "type": "model_not_found",
    "param": "model"
  }
}
```

### Errores de límite de velocidad (429)

Cuando exceda el límite de velocidad:

```json
{
  "error": {
    "message": "Rate limit exceeded. Please slow down.",
    "type": "rate_limit_error",
    "code": "rate_limit_exceeded"
  }
}
```

**Encabezados (Headers) incluidos:**

```
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 0
X-RateLimit-Reset: 1234567890
Retry-After: 60
```

### Contenido demasiado grande (413)

Cuando el tamaño de la entrada o del archivo excede el límite:

```json
{
  "error": {
    "message": "Input size exceeds maximum allowed",
    "type": "invalid_request_error",
    "code": "payload_too_large"
  }
}
```

Causas comunes:
- Archivo de imagen demasiado grande (máximo 20MB)
- Archivo de audio demasiado grande (máximo 25MB)
- El texto de entrada excede la longitud de contexto (context length) del modelo

### Errores upstream (502, 503)

| Tipo | Descripción |
|------|-------------|
| `upstream_error` | El proveedor devolvió un error |
| `all_channels_failed` | No hay proveedores disponibles |
| `timeout_error` | Tiempo de espera de la solicitud agotado |

## Manejo de errores en Python

```python
from openai import OpenAI, APIError, RateLimitError, APIConnectionError

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.lemondata.cc/v1"
)

def chat_with_retry(messages, max_retries=3):
    for attempt in range(max_retries):
        try:
            return client.chat.completions.create(
                model="gpt-4o",
                messages=messages
            )
        except RateLimitError as e:
            if attempt < max_retries - 1:
                import time
                time.sleep(2 ** attempt)  # Exponential backoff
                continue
            raise
        except APIConnectionError as e:
            print(f"Connection error: {e}")
            raise
        except APIError as e:
            print(f"API error: {e.status_code} - {e.message}")
            raise
```

## Manejo de errores en JavaScript

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: 'sk-your-api-key',
  baseURL: 'https://api.lemondata.cc/v1'
});

async function chatWithRetry(messages, maxRetries = 3) {
  for (let attempt = 0; attempt < maxRetries; attempt++) {
    try {
      return await client.chat.completions.create({
        model: 'gpt-4o',
        messages
      });
    } catch (error) {
      if (error instanceof OpenAI.RateLimitError) {
        if (attempt < maxRetries - 1) {
          await new Promise(r => setTimeout(r, 2 ** attempt * 1000));
          continue;
        }
      }
      throw error;
    }
  }
}
```

## Mejores prácticas

<AccordionGroup>
  <Accordion title="Implementar retroceso exponencial (Exponential Backoff)">
    Cuando encuentre límites de velocidad, aumente gradualmente el tiempo de espera entre reintentos:
    ```python
    wait_time = 2 ** attempt  # 1s, 2s, 4s, 8s...
    ```
  </Accordion>

  <Accordion title="Configurar tiempos de espera (Timeouts)">
    Asegúrese de configurar tiempos de espera razonables para evitar que las solicitudes se queden colgadas:
    ```python
    client = OpenAI(timeout=60.0)  # 60 秒逾時
    ```
  </Accordion>

  <Accordion title="Registrar errores para depuración">
    Registre la respuesta de error completa (incluyendo el request ID) para solicitar soporte:
    ```python
    except APIError as e:
        logger.error(f"API Error: {e.status_code} - {e.message}")
    ```
  </Accordion>

  <Accordion title="Manejar errores específicos del modelo">
    Algunos modelos tienen requisitos específicos (por ejemplo: número máximo de tokens, formato de imagen).
    Valide la entrada antes de enviar la solicitud.
  </Accordion>
</AccordionGroup>
