## Resumen

Dify es una plataforma de desarrollo de aplicaciones de LLM de código abierto. Ofrece orquestación visual de prompts, flujos de RAG, marcos de agentes y capacidades de LLMOps. LemonData puede configurarse como un proveedor de modelos personalizado en Dify.

## Ventajas

- Acceso a más de 300 modelos de AI a través de una única interfaz
- Cambia de modelo sin modificar la lógica de la aplicación
- Selecciona el mejor modelo para cada tarea para optimizar costos
- Facturación y seguimiento de uso unificados

## Requisitos previos

- Cuenta de LemonData con acceso a la API
- Instalación de Dify (versión en la nube o despliegue local)

## Pasos de configuración

### Paso 1: Obtén tu API Key

1. Inicia sesión en el [LemonData Dashboard](https://lemondata.cc/dashboard)
2. Navega a [API Keys](https://lemondata.cc/dashboard/api)
3. Crea y copia tu API Key (formato: `sk-...`)

### Paso 2: Añade un proveedor de modelos personalizado

<Steps>
  <Step title="Abrir configuración">
    En Dify, ve a **Settings** → **Model Provider**
  </Step>
  <Step title="Añadir proveedor compatible con OpenAI">
    Haz clic en **Add Model Provider** y selecciona **OpenAI-API-compatible**
  </Step>
  <Step title="Configurar proveedor">
    Ingresa la siguiente configuración:

    | Campo | Valor |
    |-------|-------|
    | Nombre del proveedor | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="Añadir modelos">
    Añade los modelos que deseas utilizar:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### Paso 3: Probar conexión

1. Selecciona un modelo (por ejemplo, `gpt-4o-mini`)
2. Envía un mensaje de prueba
3. Confirma que recibes una respuesta

## Uso en aplicaciones

### Chatbot

1. Crea una nueva aplicación de Chatbot
2. Selecciona LemonData como proveedor de modelos
3. Selecciona tu modelo preferido
4. Configura el System Prompt y los parámetros

### Agente (Agent)

1. Crea una aplicación de Agente
2. Selecciona un modelo potente (GPT-4o, Claude)
3. Añade herramientas y bases de conocimiento
4. Configura el comportamiento del Agente

### Flujo de trabajo (Workflow)

1. Crea un flujo de trabajo
2. Añade un nodo de LLM
3. Selecciona un modelo de LemonData para cada nodo
4. Conecta los nodos y configura el flujo de datos

## Modelos disponibles

| Categoría | Modelo |
|----------|--------|
| Chat | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| Embeddings | text-embedding-3-small, text-embedding-3-large |
| Visión (Vision) | GPT-4o (soporta imágenes), Claude Sonnet (soporta imágenes) |

## Configuración de RAG

Para aplicaciones de RAG, configura el modelo de embeddings:

1. Ve a **Settings** → **Model Provider**
2. Añade el modelo de embeddings: `text-embedding-3-small`
3. Establécelo como el modelo de embeddings predeterminado en la configuración de la base de conocimiento

## Mejores prácticas

<AccordionGroup>
  <Accordion title="Comienza con modelos económicos">
    Usa GPT-4o-mini durante las fases de prueba y desarrollo, y cambia a modelos más potentes en el entorno de producción.
  </Accordion>

  <Accordion title="Usa enrutamiento de modelos">
    Configura diferentes modelos para diferentes tareas: usa modelos rápidos para consultas simples y modelos potentes para razonamientos complejos.
  </Accordion>

  <Accordion title="Monitorea el uso">
    Combina las funciones de LLMOps de Dify con la consola de LemonData para realizar un seguimiento de los costos y el rendimiento.
  </Accordion>
</AccordionGroup>

## Solución de problemas

<AccordionGroup>
  <Accordion title="Error de conexión">
    - Confirma que la API Base URL sea exactamente `https://api.lemondata.cc/v1`
    - Comprueba si hay una barra diagonal al final
    - Verifica la conexión de red
  </Accordion>

  <Accordion title="401 No autorizado (Unauthorized)">
    - Vuelve a comprobar la API Key
    - Asegúrate de que la Key esté activa en la consola de LemonData
  </Accordion>

  <Accordion title="Modelo no encontrado">
    - Confirma que el nombre del modelo coincida exactamente
    - Verifica la disponibilidad del modelo en [lemondata.cc/zh-TW/models](https://lemondata.cc/zh-TW/models)
  </Accordion>
</AccordionGroup>
