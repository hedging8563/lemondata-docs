## Resumen

Google Gemini CLI es una herramienta de línea de comandos para interactuar con los modelos de Gemini. LemonData proporciona un endpoint compatible que le permite usar Gemini CLI y acceder a más de 300 modelos.

## Requisitos del sistema

- **Node.js**: Versión 20.0+
- **Sistema operativo**: Windows 10/11, macOS 10.15+, Ubuntu 20.04+, o Debian 10+

## Instalación

```bash
npm install -g @google/gemini-cli
```

Verificar la instalación:

```bash
gemini --version
```

## Configuración

### Paso 1: Obtener su API Key

1. Inicie sesión en [LemonData Dashboard](https://lemondata.cc/dashboard)
2. Navegue a [API Keys](https://lemondata.cc/dashboard/api)
3. Cree y copie su API Key (formato: `sk-...`)

### Paso 2: Configurar las variables de entorno

**Temporal (sesión actual):**

```bash
export GEMINI_API_KEY="sk-your-lemondata-key"
export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"
```

**Configuración permanente:**

Añadir a su archivo de configuración de shell:

<Tabs>
  <Tab title="Bash">
    ```bash
    echo 'export GEMINI_API_KEY="sk-your-lemondata-key"' >> ~/.bashrc
    echo 'export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"' >> ~/.bashrc
    source ~/.bashrc
    ```
  </Tab>
  <Tab title="Zsh">
    ```bash
    echo 'export GEMINI_API_KEY="sk-your-lemondata-key"' >> ~/.zshrc
    echo 'export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"' >> ~/.zshrc
    source ~/.zshrc
    ```
  </Tab>
  <Tab title="Fish">
    ```bash
    set -Ux GEMINI_API_KEY "sk-your-lemondata-key"
    set -Ux GOOGLE_GEMINI_BASE_URL "https://api.lemondata.cc"
    ```
  </Tab>
</Tabs>

<Warning>
Reinicie la terminal después de la configuración para que los cambios surtan efecto.
</Warning>

<Note>
**Limitación conocida**: En algunos casos, si existe una sesión de Google en caché, Gemini CLI podría no seguir `GOOGLE_GEMINI_BASE_URL`. Si experimenta problemas de conexión, intente limpiar la caché de la CLI o iniciar una nueva sesión de terminal.
</Note>

## Uso básico

Inicie Gemini CLI desde el directorio de su proyecto:

```bash
cd your-project
gemini
```

En la primera ejecución, deberá:
1. Seleccionar un tema
2. Confirmar los avisos de seguridad
3. Confiar en el directorio de trabajo


## Solución de problemas

<AccordionGroup>
  <Accordion title="Error de conexión">
    - Verifique que `GOOGLE_GEMINI_BASE_URL` esté configurado como `https://api.lemondata.cc`
    - Nota: El endpoint de Gemini no requiere el sufijo `/v1`
    - Compruebe la conexión a internet
  </Accordion>

  <Accordion title="Fallo de autenticación">
    - Verifique que la variable de entorno `GEMINI_API_KEY` esté configurada
    - Compruebe si la Key comienza con `sk-`
    - Asegúrese de que la Key esté activa en el panel de control (dashboard) de LemonData
  </Accordion>

  <Accordion title="Modelo no disponible">
    - Compruebe la disponibilidad de los modelos Gemini en [lemondata.cc/en/models](https://lemondata.cc/zh-TW/models)
    - Pruebe diferentes variantes del modelo Gemini
  </Accordion>
</AccordionGroup>
