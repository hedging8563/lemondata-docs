## Resumen

[Cursor](https://cursor.sh) es un editor de código impulsado por IA. Puedes usar LemonData como un proveedor de API personalizado para acceder a más de 300 modelos.

## Configuración

1. Abre los Ajustes de Cursor (`Cmd/Ctrl + ,`)
2. Navega a **Models** en la barra lateral izquierda
3. Desplázate hacia abajo hasta la sección **OpenAI API Key**
4. Introduce tu clave API de LemonData
5. Activa **Override OpenAI Base URL**
6. Establece la Base URL a:

```
https://api.lemondata.cc/v1
```

## Selección de Modelos

Después de la configuración, puedes seleccionar cualquier modelo de LemonData en Cursor:

### Modelos Recomendados para Programación

| Modelo | Ideal para |
|-------|----------|
| `claude-opus-4-5` | Programación compleja, arquitectura |
| `claude-sonnet-4-5` | Programación general, depuración |
| `gpt-4o` | Generación de código, explicaciones |
| `deepseek-r1` | Tareas con alta carga de razonamiento |
| `gpt-4o-mini` | Completados rápidos, tareas simples |

## Características

### Chat

Usa el panel de chat de IA (`Cmd/Ctrl + L`) con cualquier modelo de LemonData:

- Explicaciones de código
- Corrección de errores
- Sugerencias de refactorización
- Generación de documentación

### Composer

La función Composer (`Cmd/Ctrl + I`) funciona con los modelos de LemonData para:

- Ediciones de múltiples archivos
- Implementación de funciones
- Migraciones de código

### Completado con tabulador

Para los completados con tabulador, recomendamos usar modelos rápidos:

- `gpt-4o-mini`
- `claude-haiku-4-5`
- `gemini-2.5-flash`

## Solución de Problemas

<AccordionGroup>
  <Accordion title="Los modelos no aparecen">
    Cursor almacena en caché las listas de modelos. Intenta reiniciar Cursor después de configurar la API.
  </Accordion>

  <Accordion title="Respuestas lentas">
    Verifica tu selección de modelos. Usa modelos más rápidos como `gpt-4o-mini` para tareas rápidas.
  </Accordion>

  <Accordion title="Errores de autenticación">
    Verifica que tu clave API sea correcta y tenga saldo suficiente.
  </Accordion>
</AccordionGroup>
