---
title: "Crear Chat Completion"
openapi: "POST /v1/chat/completions"
description: "Crea una completación para el mensaje de chat"
---

## Autorización

<ParamField header="Authorization" type="string" required>
  Token Bearer para autenticación de API. Formato: `Bearer sk-your-api-key`
</ParamField>

## Cuerpo de la solicitud

<ParamField body="model" type="string" required>
  ID del modelo a utilizar. Consulta [Modelos](https://lemondata.cc/es/models) para ver las opciones disponibles.
</ParamField>

<ParamField body="messages" type="array" required>
  Una lista de mensajes que componen la conversación.

  Cada objeto de mensaje contiene:
  - `role` (string): `system`, `user`, o `assistant`
  - `content` (string | array): El contenido del mensaje
</ParamField>

<ParamField body="temperature" type="number" default="1">
  Temperatura de muestreo entre 0 y 2. Valores más altos hacen que el resultado sea más aleatorio.
</ParamField>

<ParamField body="max_tokens" type="integer">
  Número máximo de tokens a generar.
</ParamField>

<ParamField body="stream" type="boolean" default="false">
  Si es true, se enviarán deltas de mensajes parciales como eventos SSE.
</ParamField>

<ParamField body="stream_options" type="object">
  Opciones para streaming. Establece `include_usage: true` para recibir el uso de tokens en fragmentos de stream.
</ParamField>

<ParamField body="top_p" type="number" default="1">
  Parámetro de muestreo de núcleo. Recomendamos alterar este o la temperatura, pero no ambos.
</ParamField>

<ParamField body="frequency_penalty" type="number" default="0">
  Número entre -2.0 y 2.0. Los valores positivos penalizan los tokens repetidos.
</ParamField>

<ParamField body="presence_penalty" type="number" default="0">
  Número entre -2.0 y 2.0. Los valores positivos penalizan los tokens que ya están en el texto.
</ParamField>

<ParamField body="stop" type="string | array">
  Hasta 4 secuencias donde la API dejará de generar tokens.
</ParamField>

<ParamField body="tools" type="array">
  Una lista de herramientas que el modelo puede llamar (function calling).
</ParamField>

<ParamField body="tool_choice" type="string | object">
  Controla cómo el modelo utiliza las herramientas. Opciones: `auto`, `none`, `required`, o un objeto de herramienta específico.
</ParamField>

<ParamField body="parallel_tool_calls" type="boolean" default="true">
  Indica si se debe habilitar la llamada a funciones en paralelo. Establécelo en false para llamar a las funciones de forma secuencial.
</ParamField>
