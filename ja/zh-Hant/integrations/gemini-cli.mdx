## 概要

Google Gemini CLI は、Gemini モデルと対話するためのコマンドラインツールです。LemonData は互換性のあるエンドポイントを提供しており、Gemini CLI を使用して 300 以上のモデルにアクセスできます。

## システム要件

- **Node.js**: バージョン 20.0+
- **オペレーティングシステム**: Windows 10/11, macOS 10.15+, Ubuntu 20.04+, または Debian 10+

## インストール

```bash
npm install -g @google/gemini-cli
```

インストールの確認：

```bash
gemini --version
```

## 設定

### ステップ 1：API Key の取得

1. [LemonData Dashboard](https://lemondata.cc/dashboard) にログインします
2. [API Keys](https://lemondata.cc/dashboard/api) に移動します
3. API Key を作成してコピーします（形式：`sk-...`）

### ステップ 2：環境変数の設定

**一時的（現在のセッション）：**

```bash
export GEMINI_API_KEY="sk-your-lemondata-key"
export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"
```

**永続的な設定：**

シェル設定ファイルに追加します：

<Tabs>
  <Tab title="Bash">
    ```bash
    echo 'export GEMINI_API_KEY="sk-your-lemondata-key"' >> ~/.bashrc
    echo 'export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"' >> ~/.bashrc
    source ~/.bashrc
    ```
  </Tab>
  <Tab title="Zsh">
    ```bash
    echo 'export GEMINI_API_KEY="sk-your-lemondata-key"' >> ~/.zshrc
    echo 'export GOOGLE_GEMINI_BASE_URL="https://api.lemondata.cc"' >> ~/.zshrc
    source ~/.zshrc
    ```
  </Tab>
  <Tab title="Fish">
    ```bash
    set -Ux GEMINI_API_KEY "sk-your-lemondata-key"
    set -Ux GOOGLE_GEMINI_BASE_URL "https://api.lemondata.cc"
    ```
  </Tab>
</Tabs>

<Warning>
設定完了後、変更を有効にするためにターミナルを再起動してください。
</Warning>

<Note>
**既知の制限**：特定の状況下で、キャッシュされた Google セッションが存在する場合、Gemini CLI が `GOOGLE_GEMINI_BASE_URL` を無視することがあります。接続の問題が発生した場合は、CLI キャッシュをクリアするか、新しいターミナルセッションを開始してください。
</Note>

## 基本的な使い方

プロジェクトディレクトリから Gemini CLI を起動します：

```bash
cd your-project
gemini
```

初回実行時には、以下の操作が必要です：
1. テーマの選択
2. 安全に関する注意事項の確認
3. 作業ディレクトリの信頼

## 利用可能なモデル

| モデル | 説明 |
|-------|-------------|
| `gemini-2.5-pro` | 最も強力な Gemini モデル |
| `gemini-2.5-flash` | 高速かつ効率的で、ほとんどのタスクに適しています |
| `gemini-2.0-flash` | 前世代のバージョン、安定版 |

## トラブルシューティング

<AccordionGroup>
  <Accordion title="接続エラー">
    - `GOOGLE_GEMINI_BASE_URL` が `https://api.lemondata.cc` に設定されていることを確認してください
    - 注意：Gemini エンドポイントには `/v1` サフィックスは不要です
    - ネットワーク接続を確認してください
  </Accordion>

  <Accordion title="認証失敗">
    - `GEMINI_API_KEY` 環境変数が設定されていることを確認してください
    - Key が `sk-` で始まっているか確認してください
    - LemonData ダッシュボードでその Key が有効であることを確認してください
  </Accordion>

  <Accordion title="モデルが利用不可">
    - [lemondata.cc/en/models](https://lemondata.cc/zh-TW/models) で Gemini モデルの利用可能性を確認してください
    - 別の Gemini モデルバリアントを試してください
  </Accordion>
</AccordionGroup>
