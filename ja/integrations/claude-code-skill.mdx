## このスキルでできること

コード内で AI 機能（GPT-4、画像生成、音声認識など）を使用したい場合、このスキルは以下のことを行います：

1. ✅ **API を直接試行** — 検索不要
2. ✅ **エラーから自動修正** — 構造化されたエラーヒントが次の試行をガイド
3. ✅ お好みの言語で**完全で実行可能なコードを生成**
4. ✅ リアルタイムの価格と可用性に基づいて**最適なモデルを選択**
5. ✅ ネイティブエンドポイントの提案で**統合を最適化**

**要約：ニーズを伝えるだけで、動作する API 統合コードを取得できます — 最初の試行が完璧でなくても、API が Claude を正しい答えに導きます。**

### 例 1：GPT-4 を使用する

**ユーザー：**
```
I want to use GPT-4 in my Python project
```

**Claude Code の動作：**
1. 最適なモデル推測で API を直接呼び出す
2. モデル名が間違っていた場合、エラーから `did_you_mean` を読み取り自動修正
3. 正しいモデルで完全な Python コードを生成

**取得結果：**
```python
from openai import OpenAI

client = OpenAI(
    api_key="sk-your-api-key",
    base_url="https://api.lemondata.cc/v1"
)

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "Hello!"}]
)

print(response.choices[0].message.content)
```

## Agent-First API の特徴

LemonData の API は AI エージェント向けに設計されています。すべてのエラーレスポンスに構造化されたヒントが含まれます：

### スマートエラーリカバリー

| エラー | API が返す内容 | Claude の動作 |
|-------|---------------------|--------------------|
| モデル名の誤り | `did_you_mean` + `suggestions` | 自動修正してリトライ |
| 残高不足 | `balance_usd` + より安い `suggestions` | 手頃なモデルに切り替え |
| モデル利用不可 | `alternatives` + `retry_after` | 利用可能なモデルに切り替え |
| レート制限 | `retry_after`（正確な秒数） | 待機してリトライ |
| コンテキスト超過 | より大きなモデルの `suggestions` | より大きなコンテキストのモデルに切り替え |

### モデルディスカバリー

ドキュメントを検索する必要はありません。以下の API エンドポイントを使用してください：

```bash
# 機械可読な API 概要を読む
curl https://lemondata.cc/llms.txt

# カテゴリ別にモデルを一覧表示
curl "https://api.lemondata.cc/v1/models?category=chat" -H "Authorization: Bearer sk-KEY"
curl "https://api.lemondata.cc/v1/models?category=image" -H "Authorization: Bearer sk-KEY"

# 機能でフィルタリング
curl "https://api.lemondata.cc/v1/models?tag=coding" -H "Authorization: Bearer sk-KEY"
```

### ネイティブエンドポイント最適化

Claude が `/v1/chat/completions` で Claude または Gemini モデルを使用すると、API が最適化ヘッダーを返します：

```
X-LemonData-Hint: This model supports native Anthropic format. Use POST /v1/messages for better performance.
X-LemonData-Native-Endpoint: /v1/messages
```

Claude Code は自動的にネイティブエンドポイントに切り替えて、パフォーマンスを向上させます。

## 重要な注意事項

### API Key のセキュリティ

<CardGroup cols={2}>
  <Card title="❌ 絶対にしないでください" icon="xmark" color="#ef4444">
    - フロントエンドコードに API Key を直接記述する
    - GitHub に API Key をコミットする
    - API Key を他人に共有する
  </Card>
  <Card title="✅ ベストプラクティス" icon="check" color="#22c55e">
    - 環境変数を使用する
    - Web サイトの場合はバックエンドから API を呼び出す
    - Next.js や Express などのフレームワークを使用する
  </Card>
</CardGroup>

### コスト管理

- 各 API 呼び出しでクレジットが消費されます
- [ダッシュボード](https://lemondata.cc/dashboard)で利用制限を設定してください
- テストには少量のデータを使用してください
- `error.balance_usd` と `error.estimated_cost_usd` でプログラム的にコストを監視できます

### エラー処理

生成されたコードには Agent-First エラー処理が含まれています：
- 構造化されたエラーヒント（`did_you_mean`、`suggestions`、`retry_after`）を読み取る
- 一般的なエラーから自動回復
- 必要に応じて代替モデルにフォールバック
