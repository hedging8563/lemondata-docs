## Gambaran Umum

Dify adalah platform pengembangan aplikasi LLM sumber terbuka. Dify menyediakan orkestrasi prompt visual, pipeline RAG, framework agen, dan kapabilitas LLMOps. LemonData dapat dikonfigurasi sebagai penyedia model kustom di Dify.

## Manfaat

- Akses 300+ model AI melalui satu antarmuka
- Beralih antar model tanpa mengubah logika aplikasi
- Optimalkan biaya dengan memilih model terbaik untuk setiap tugas
- Penagihan dan pelacakan penggunaan yang terpadu

## Prasyarat

- Akun LemonData dengan akses API
- Instalasi Dify (cloud atau self-hosted)

## Langkah Konfigurasi

### Langkah 1: Dapatkan API Key Anda

1. Masuk ke [LemonData Dashboard](https://lemondata.cc/dashboard)
2. Navigasi ke [API Keys](https://lemondata.cc/dashboard/api)
3. Buat dan salin API key Anda (format: `sk-...`)

### Langkah 2: Tambahkan Penyedia Model Kustom

<Steps>
  <Step title="Buka Pengaturan">
    Di Dify, buka **Settings** → **Model Provider**
  </Step>
  <Step title="Tambahkan Penyedia Kompatibel OpenAI">
    Klik **Add Model Provider** dan pilih **OpenAI-API-compatible**
  </Step>
  <Step title="Konfigurasi Penyedia">
    Masukkan pengaturan berikut:

    | Bidang | Nilai |
    |-------|-------|
    | Provider Name | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="Tambahkan Model">
    Tambahkan model yang ingin Anda gunakan:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### Langkah 3: Uji Koneksi

1. Pilih sebuah model (misalnya, `gpt-4o-mini`)
2. Kirim pesan uji coba
3. Verifikasi bahwa Anda menerima respons

## Penggunaan dalam Aplikasi

### Chatbot

1. Buat aplikasi Chatbot baru
2. Pilih LemonData sebagai penyedia model
3. Pilih model pilihan Anda
4. Konfigurasi prompt sistem dan parameter

### Agen

1. Buat aplikasi Agen
2. Pilih model yang mumpuni (GPT-4o, Claude)
3. Tambahkan alat dan basis pengetahuan
4. Konfigurasi perilaku agen

### Alur Kerja

1. Buat Alur Kerja (Workflow)
2. Tambahkan node LLM
3. Pilih model LemonData untuk setiap node
4. Hubungkan node dan konfigurasi aliran data

## Model yang Tersedia

| Kategori | Model |
|----------|--------|
| Chat | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| Embedding | text-embedding-3-small, text-embedding-3-large |
| Visi | GPT-4o (dengan gambar), Claude Sonnet (dengan gambar) |

## Konfigurasi RAG

Untuk aplikasi RAG, konfigurasi embedding:

1. Buka **Settings** → **Model Provider**
2. Tambahkan model embedding: `text-embedding-3-small`
3. Atur sebagai model embedding default dalam pengaturan basis pengetahuan

## Praktik Terbaik

<AccordionGroup>
  <Accordion title="Mulai dengan model ekonomis">
    Gunakan GPT-4o-mini untuk pengujian dan pengembangan, beralih ke model yang lebih kuat untuk produksi.
  </Accordion>

  <Accordion title="Gunakan perutean model">
    Konfigurasi model yang berbeda untuk tugas yang berbeda - model cepat untuk kueri sederhana, model kuat untuk penalaran kompleks.
  </Accordion>

  <Accordion title="Pantau penggunaan">
    Gunakan fitur LLMOps Dify bersama dengan dashboard LemonData untuk melacak biaya dan performa.
  </Accordion>
</AccordionGroup>

## Penyelesaian Masalah

<AccordionGroup>
  <Accordion title="Kesalahan Koneksi">
    - Verifikasi API Base URL tepat `https://api.lemondata.cc/v1`
    - Periksa adanya garis miring di akhir (trailing slashes)
    - Verifikasi konektivitas jaringan
  </Accordion>

  <Accordion title="401 Unauthorized">
    - Periksa kembali API key
    - Pastikan kunci aktif di dashboard LemonData
  </Accordion>

  <Accordion title="Model Tidak Ditemukan">
    - Verifikasi nama model cocok secara tepat
    - Periksa ketersediaan model di [lemondata.cc/en/models](https://lemondata.cc/id/models)
  </Accordion>
</AccordionGroup>
