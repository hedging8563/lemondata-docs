## Fitur API Agent-First

API LemonData dirancang untuk agen AI. Setiap respons error menyertakan petunjuk terstruktur:

### Pemulihan Error Cerdas

| Error | Apa yang Dikembalikan API | Apa yang Dilakukan Claude |
|-------|--------------------------|--------------------------|
| Nama model salah | `did_you_mean` + `suggestions` | Memperbaiki otomatis dan mencoba ulang |
| Saldo tidak cukup | `balance_usd` + `suggestions` yang lebih murah | Beralih ke model yang terjangkau |
| Model tidak tersedia | `alternatives` + `retry_after` | Beralih ke model yang tersedia |
| Rate limited | `retry_after` (detik yang tepat) | Menunggu dan mencoba ulang |
| Konteks terlalu panjang | `suggestions` dengan model yang lebih besar | Beralih ke model dengan konteks lebih besar |

### Penemuan Model

Tidak perlu mencari dokumentasi. Gunakan API endpoint berikut:

```bash
# Baca ringkasan API yang dapat dibaca mesin
curl https://lemondata.cc/llms.txt

# Daftar model berdasarkan kategori
curl "https://api.lemondata.cc/v1/models?category=chat" -H "Authorization: Bearer sk-KEY"
curl "https://api.lemondata.cc/v1/models?category=image" -H "Authorization: Bearer sk-KEY"

# Filter berdasarkan kemampuan
curl "https://api.lemondata.cc/v1/models?tag=coding" -H "Authorization: Bearer sk-KEY"
```

### Optimasi Native Endpoint

Ketika Claude menggunakan `/v1/chat/completions` dengan model Claude atau Gemini, API mengembalikan header optimasi:

```
X-LemonData-Hint: This model supports native Anthropic format. Use POST /v1/messages for better performance.
X-LemonData-Native-Endpoint: /v1/messages
```

Claude Code akan secara otomatis beralih ke native endpoint untuk performa yang lebih baik.

### Manajemen Biaya

- Setiap panggilan API menghabiskan kredit
- Atur batas penggunaan di [dashboard](https://lemondata.cc/dashboard) Anda
- Gunakan volume data kecil untuk pengujian
- Periksa `error.balance_usd` dan `error.estimated_cost_usd` untuk memantau biaya secara programatis

### Penanganan Error

Kode yang dihasilkan mencakup penanganan error Agent-First yang:
- Membaca petunjuk error terstruktur (`did_you_mean`, `suggestions`, `retry_after`)
- Memulihkan secara otomatis dari error umum
- Beralih ke model alternatif saat diperlukan
