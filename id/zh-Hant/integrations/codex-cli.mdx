## Ikhtisar

OpenAI Codex adalah alat baris perintah (CLI) sumber terbuka yang berfungsi sebagai agen pengodean (coding agent) ringan, yang mampu membaca, memodifikasi, dan mengeksekusi kode di terminal. Alat ini dibangun berdasarkan model GPT dan dioptimalkan untuk pembuatan kode.

## Persyaratan Sistem

- **Sistem Operasi**: macOS, Linux (dukungan resmi), Windows melalui WSL
- **Node.js**: Versi 18+
- **npm**: Versi 10.x.x atau lebih tinggi

## Instalasi

```bash
sudo npm install -g @openai/codex@latest
```

Verifikasi instalasi:

```bash
codex --version
```

## Konfigurasi

### Langkah 1: Mengatur API Key

**Sementara (sesi saat ini):**

```bash
export OPENAI_API_KEY="sk-your-lemondata-key"
```

**Konfigurasi permanen:**

Tambahkan ke `~/.bashrc`, `~/.zshrc`, atau `~/.bash_profile`:

```bash
export OPENAI_API_KEY="sk-your-lemondata-key"
```

Kemudian muat ulang:

```bash
source ~/.zshrc  # atau source ~/.bashrc
```

### Langkah 2: Konfigurasi config.toml

Edit `~/.codex/config.toml`:

```toml
model = "gpt-4o"
model_provider = "lemondata"

[model_providers.lemondata]
name = "LemonData"
base_url = "https://api.lemondata.cc/v1"
env_key = "OPENAI_API_KEY"
wire_api = "chat"
```

<Note>
Jika file konfigurasi tidak ada, jalankan `codex` sekali untuk membuat file, lalu lakukan pengeditan.
</Note>

## Penggunaan Dasar

**Mulai mode interaktif:**

```bash
codex
```

**Instruksi langsung:**

```bash
codex "Fix the bug in main.py line 42"
```

**Tentukan model:**

```bash
codex -m gpt-4o "Build a REST API server"
```

## Model yang Tersedia

| Model | Skenario Penggunaan |
|-------|----------|
| `gpt-4o` | Tugas pengodean kompleks, desain arsitektur |
| `gpt-4o-mini` | Perbaikan cepat, tugas sederhana |
| `claude-sonnet-4-5` | Peninjauan kode, penulisan dokumentasi |
| `deepseek-r1` | Desain algoritma, penalaran |

## Perintah Interaktif

| Perintah | Deskripsi |
|---------|-------------|
| `/help` | Tampilkan bantuan |
| `/exit` atau `Ctrl+C` | Keluar |
| `/clear` | Bersihkan percakapan |
| `/config` | Lihat konfigurasi |
| `/model <name>` | Ganti model |
| `/tokens` | Lihat penggunaan token |

## Verifikasi Konfigurasi

```bash
# Periksa variabel lingkungan
echo $OPENAI_API_KEY

# Uji koneksi API
codex "Hello, Codex!"

# Lihat konfigurasi
cat ~/.codex/config.toml
```

## Kasus Penggunaan Umum

**Peninjauan kode:**

```bash
git diff | codex "Review these code changes"
```

**Hasilkan pesan commit:**

```bash
git diff --staged | codex "Generate a commit message for these changes"
```

**Perbaiki kesalahan:**

```bash
codex "Fix the TypeScript errors in src/components/"
```

**Jelaskan kode:**

```bash
cat main.py | codex "Explain what this code does"
```

## Penyelesaian Masalah

<AccordionGroup>
  <Accordion title="Kesalahan Koneksi">
    - Verifikasi apakah `base_url` di config.toml sudah tepat `https://api.lemondata.cc/v1`
    - Periksa status koneksi jaringan
    - Pastikan tidak ada gangguan server proksi
  </Accordion>

  <Accordion title="Autentikasi Gagal">
    - Verifikasi bahwa variabel lingkungan `OPENAI_API_KEY` telah diatur
    - Periksa apakah kunci dimulai dengan `sk-`
    - Pastikan kunci dalam status aktif di dasbor LemonData
  </Accordion>

  <Accordion title="Model Tidak Ditemukan">
    - Periksa apakah nama model sudah sesuai sepenuhnya
    - Verifikasi ketersediaan model di [lemondata.cc/en/models](https://lemondata.cc/zh-TW/models)
  </Accordion>
</AccordionGroup>
