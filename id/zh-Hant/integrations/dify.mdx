## Ikhtisar

Dify adalah platform pengembangan aplikasi LLM sumber terbuka. Platform ini menyediakan orkestrasi Prompt visual, alur kerja RAG, framework Agent, serta kemampuan LLMOps. LemonData dapat dikonfigurasi sebagai penyedia model kustom di dalam Dify.

## Keuntungan

- Akses ke 300+ model AI melalui satu antarmuka tunggal
- Beralih antar model tanpa perlu mengubah logika aplikasi
- Pilih model terbaik untuk setiap tugas guna mengoptimalkan biaya
- Penagihan dan pelacakan penggunaan yang terpadu

## Prasyarat

- Akun LemonData dengan hak akses API
- Instalasi Dify (versi cloud atau deployment mandiri)

## Langkah Konfigurasi

### Langkah 1: Dapatkan API Key Anda

1. Login ke [LemonData Dashboard](https://lemondata.cc/dashboard)
2. Navigasi ke [API Keys](https://lemondata.cc/dashboard/api)
3. Buat dan salin API Key Anda (format: `sk-...`)

### Langkah 2: Tambahkan Penyedia Model Kustom

<Steps>
  <Step title="Buka Pengaturan">
    Di Dify, buka **Settings** → **Model Provider**
  </Step>
  <Step title="Tambahkan Penyedia Kompatibel OpenAI">
    Klik **Add Model Provider** dan pilih **OpenAI-API-compatible**
  </Step>
  <Step title="Konfigurasi Penyedia">
    Masukkan pengaturan berikut:

    | Bidang | Nilai |
    |-------|-------|
    | Nama Penyedia | LemonData |
    | API Key | `sk-your-lemondata-key` |
    | API Base URL | `https://api.lemondata.cc/v1` |
  </Step>
  <Step title="Tambahkan Model">
    Tambahkan model yang ingin Anda gunakan:
    - `gpt-4o`
    - `gpt-4o-mini`
    - `claude-sonnet-4-5`
    - `claude-opus-4-5`
    - `gemini-2.5-flash`
    - `gemini-2.5-pro`
    - `deepseek-r1`
  </Step>
</Steps>

### Langkah 3: Uji Koneksi

1. Pilih sebuah model (misalnya `gpt-4o-mini`)
2. Kirim pesan uji coba
3. Konfirmasi bahwa Anda menerima balasan

## Penggunaan dalam Aplikasi

### Chatbot

1. Buat aplikasi Chatbot baru
2. Pilih LemonData sebagai penyedia model
3. Pilih model yang Anda inginkan
4. Konfigurasi System Prompt dan parameter

### Agent

1. Buat aplikasi Agent baru
2. Pilih model yang berkemampuan tinggi (GPT-4o, Claude)
3. Tambahkan alat dan basis pengetahuan
4. Konfigurasi perilaku Agent

### Workflow

1. Buat sebuah workflow
2. Tambahkan node LLM
3. Pilih model LemonData untuk setiap node
4. Hubungkan node dan konfigurasi aliran data

## Model yang Tersedia

| Kategori | Model |
|----------|--------|
| Percakapan (Chat) | GPT-4o, GPT-4o-mini, Claude Sonnet/Opus, Gemini, DeepSeek |
| Embedding (Embeddings) | text-embedding-3-small, text-embedding-3-large |
| Visi (Vision) | GPT-4o (mendukung gambar), Claude Sonnet (mendukung gambar) |

## Konfigurasi RAG

Untuk aplikasi RAG, silakan konfigurasi model embedding:

1. Buka **Settings** → **Model Provider**
2. Tambahkan model embedding: `text-embedding-3-small`
3. Atur sebagai model embedding default dalam pengaturan basis pengetahuan

## Praktik Terbaik

<AccordionGroup>
  <Accordion title="Mulai dengan Model Ekonomis">
    Gunakan GPT-4o-mini selama fase pengujian dan pengembangan, lalu beralih ke model yang lebih kuat di lingkungan produksi.
  </Accordion>

  <Accordion title="Gunakan Perutean Model">
    Konfigurasi model yang berbeda untuk tugas yang berbeda — gunakan model cepat untuk kueri sederhana, dan model kuat untuk penalaran kompleks.
  </Accordion>

  <Accordion title="Pantau Penggunaan">
    Gunakan fitur LLMOps Dify bersama dengan konsol LemonData untuk melacak biaya dan performa.
  </Accordion>
</AccordionGroup>

## Pemecahan Masalah

<AccordionGroup>
  <Accordion title="Kesalahan Koneksi">
    - Pastikan API Base URL tepat `https://api.lemondata.cc/v1`
    - Periksa apakah ada garis miring di bagian akhir
    - Periksa koneksi internet
  </Accordion>

  <Accordion title="401 Tidak Terotorisasi (Unauthorized)">
    - Periksa kembali API Key
    - Pastikan Key tersebut dalam status aktif di konsol LemonData
  </Accordion>

  <Accordion title="Model Tidak Ditemukan">
    - Pastikan nama model cocok sepenuhnya
    - Periksa ketersediaan model di [lemondata.cc/en/models](https://lemondata.cc/zh-TW/models)
  </Accordion>
</AccordionGroup>
