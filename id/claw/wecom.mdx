Integrasikan WeCom (企业微信 / WeChat Work) dengan instansi Claw Anda untuk menghadirkan AI ke ruang kerja perusahaan Anda.

## Prasyarat

- Akun admin [WeCom](https://work.weixin.qq.com/)
- Sebuah instansi Claw ([buat di sini](https://lemondata.cc/dashboard/claw))

## Panduan Penyiapan

<Steps>
  <Step title="Buat Aplikasi WeCom">
    1. Masuk ke [Konsol Admin WeCom](https://work.weixin.qq.com/wework_admin/frame)
    2. Buka **App Management** → **Create App**
    3. Masukkan nama aplikasi, unggah logo, dan atur rentang visibilitas
    4. Klik **Create**
  </Step>
  <Step title="Dapatkan Kredensial">
    1. Catat **Corp ID** Anda — ditemukan di **My Enterprise** → **Enterprise Info** di bagian bawah
    2. Di pengaturan aplikasi Anda, salin **Secret** (Corp Secret)
  </Step>
  <Step title="Konfigurasi URL Callback">
    1. Di pengaturan aplikasi Anda, buka **Receive Messages** → **Set API Receive**
    2. URL callback Anda ditampilkan di dasbor LemonData saat Anda memperluas bagian WeCom:
    ```
    https://claw-{your-subdomain}.lemondata.cc/webhook/wecom
    ```
    3. Atur **Token** (string acak) dan **EncodingAESKey** (string acak 43 karakter)
    4. Klik **Save** — WeCom akan memverifikasi URL callback

    <Tip>
    URL callback ditampilkan di dasbor LemonData dengan tombol salin. Pastikan instansi Claw Anda sedang berjalan sebelum memverifikasi URL.
    </Tip>
  </Step>
  <Step title="Konfigurasi di Dasbor LemonData">
    1. Buka [Dasbor Claw](https://lemondata.cc/dashboard/claw) Anda
    2. Klik **ikon pengaturan** pada instansi Anda
    3. Buka tab **IM Integrations**
    4. Perluas bagian **WeCom**
    5. Salin **Callback URL** yang ditampilkan di bagian atas
    6. Isi: **Corp ID**, **Corp Secret**, **Token**, **EncodingAESKey**
    7. Klik **Save**
  </Step>
  <Step title="Mulai Mengobrol">
    1. Di WeCom, temukan aplikasi Anda di **Workbench**
    2. Kirim pesan ke aplikasi
    3. AI akan merespons secara langsung — tidak perlu penyandingan untuk WeCom
  </Step>
</Steps>

<Warning>
**Token** dan **EncodingAESKey** harus sama persis antara konsol admin WeCom dan konfigurasi Claw Anda. Ketidakcocokan akan menyebabkan verifikasi pesan gagal.
</Warning>

## Penyelesaian Masalah

<AccordionGroup>
  <Accordion title="Verifikasi URL Callback gagal">
    - Pastikan instansi Claw Anda sedang berjalan (status hijau)
    - Verifikasi format URL: `https://claw-{subdomain}.lemondata.cc/webhook/wecom`
    - Periksa apakah Token dan EncodingAESKey sudah disimpan di konfigurasi Claw **sebelum** melakukan verifikasi di WeCom
  </Accordion>
  <Accordion title="Bot tidak merespons">
    - Verifikasi bahwa Corp ID and Corp Secret sudah benar
    - Periksa apakah aplikasi terlihat oleh akun pengguna Anda (pengaturan rentang visibilitas)
    - Pastikan Token dan EncodingAESKey cocok antara WeCom dan Claw
  </Accordion>
</AccordionGroup>
