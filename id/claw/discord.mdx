Hubungkan instansi Claw Anda ke Discord dan mengobrol dengan AI secara langsung di server atau DM Anda.

## Prasyarat

- Akun [Discord](https://discord.com/)
- Server Discord di mana Anda memiliki izin **Manage Server**
- Instansi Claw ([buat di sini](https://lemondata.cc/dashboard/claw))

## Panduan Pengaturan

<Steps>
  <Step title="Buat Aplikasi Discord">
    1. Buka [Discord Developer Portal](https://discord.com/developers/applications)
    2. Klik **New Application**
    3. Berikan nama (misal: "My AI Bot") dan klik **Create**
  </Step>
  <Step title="Buat Bot">
    1. Di dalam aplikasi Anda, buka tab **Bot**
    2. Klik **Add Bot** → **Yes, do it!**
    3. Di bawah **Privileged Gateway Intents**, aktifkan:
       - **Message Content Intent** (diperlukan)
       - **Server Members Intent** (disarankan)
    4. Klik **Reset Token** untuk membuat token bot
    5. Salin token tersebut — Anda akan membutuhkannya nanti
    <Warning>
    Token hanya ditampilkan sekali. Jika Anda menghilangkannya, Anda perlu melakukan reset.
    </Warning>
  </Step>
  <Step title="Undang Bot ke Server Anda">
    1. Buka tab **OAuth2** → **URL Generator**
    2. Di bawah **Scopes**, pilih: `bot`, `applications.commands`
    3. Di bawah **Bot Permissions**, pilih: `Send Messages`, `Read Message History`
    4. Salin URL yang dihasilkan dan buka di browser Anda
    5. Pilih server Anda dan klik **Authorize**
  </Step>
  <Step title="Konfigurasi di Dashboard LemonData">
    1. Buka [Claw Dashboard](https://lemondata.cc/dashboard/claw) Anda
    2. Klik **ikon pengaturan** pada instansi Anda
    3. Buka tab **IM Integrations**
    4. Ekspansi bagian **Discord**
    5. Tempel **Bot Token** Anda
    6. Klik **Save**
  </Step>
  <Step title="Mulai Mengobrol">
    1. Di server Discord Anda, mention bot atau kirimkan DM
    2. Pengguna pertama secara **otomatis dipasangkan** sebagai pemilik — tidak memerlukan kode!
    3. Mulai mengobrol dengan AI!
  </Step>
</Steps>

## Penggunaan di Server

### Pengaturan Grup

Konfigurasikan perilaku server (guild) di tab **IM Integrations** pada Dashboard Anda:

| Pengaturan | Opsi | Deskripsi |
|---------|---------|-------------|
| **Group Policy** | Open (default) / Allowlist / Disabled | Mengontrol akses ke saluran server |
| **Require @mention** | On / Off | Bot hanya merespons saat di-@mention (disarankan) |
| **Group IDs** | Daftar ID guild | Batasi ke server tertentu (kosong = semua server) |

<Note>
Discord menggunakan allowlist pengguna per-guild alih-alih allowlist grup global. Untuk pemfilteran pengguna per-guild tingkat lanjut, konfigurasikan secara langsung di file konfigurasi OpenClaw.
</Note>

### Kebijakan DM

Kontrol siapa yang dapat mengirim pesan langsung ke bot Anda:

| Kebijakan | Perilaku |
|--------|----------|
| **Pairing** (default) | Pengguna baru mendapatkan kode pairing — masukkan di WebUI untuk mengotorisasi |
| **Allowlist** | Hanya ID pengguna yang Anda tentukan yang dapat mengirim DM ke bot |
| **Open** | Siapa pun dapat mengirim DM ke bot (gunakan dengan hati-hati) |

## Pemecahan Masalah

<AccordionGroup>
  <Accordion title="Bot sedang online tetapi tidak merespons">
    - Pastikan **Message Content Intent** diaktifkan di Developer Portal
    - Verifikasi bahwa token bot sudah benar
    - Periksa apakah bot memiliki izin untuk membaca dan mengirim pesan di saluran tersebut
  </Accordion>
  <Accordion title="Bot terlihat offline">
    - Periksa status instansi Anda di dashboard (seharusnya berwarna hijau/running)
    - Coba mulai ulang instansi
    - Verifikasi bahwa token belum di-reset di Developer Portal
  </Accordion>
  <Accordion title="Kesalahan 409 Conflict di log">
    - Hal ini terjadi ketika dua instansi mencoba menggunakan token bot yang sama
    - Pastikan hanya satu instansi Claw yang dikonfigurasi dengan token ini
  </Accordion>
</AccordionGroup>
